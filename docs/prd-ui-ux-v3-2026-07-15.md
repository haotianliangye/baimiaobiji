# 白描笔记 UI/UX 重构 PRD v3（需求补漏）

> 日期：2026-07-15
> 基于会话需求整理
> 取代/补充：`docs/prd-ui-ux-2026-07-14.md`
> 状态：ready-for-agent

---

## Problem Statement

在白描笔记 v2 UI/UX 重构落地后，用户在真实使用过程中发现以下体验断层：

1. **拾微/回顾内容展示过长**：中间内容区文字过多时会占满甚至跨越多屏，浏览效率低；需要按行数折叠，并保留双击编辑、长按菜单等旧交互。
2. **设置页“界面语言”独立模块冗余**：当前语言切换被做成一个独立卡片模块，占据过多空间，应并入“对话模型”上方作为平铺选项。
3. **设置抽屉遮罩过实**：抽屉展开后用灰色实底遮罩挡住后方应用版面，视觉上像跳到了新页面，应改为毛玻璃透底效果。
4. **抽屉“所有标签”区缺少收纳与管理入口**：“所有标签”无法收起，管理入口文字过长；应支持展开/收起，并用图标替代文字入口。
5. **标签缺少快捷操作菜单**：现有标签只有查看入口，缺少置顶、编辑、移除/删除等高频操作。
6. **日历弹窗统计口径未随数据合并更新**：V2 已将“日记”合并进“回顾”，但日历弹窗仍显示“日记”并只统计旧日记数量。
7. **Copilot RAG 选择器文案与索引未对齐新四类**：当前 RAG 选择器仍是“记录/日记/回顾/洞察”，需与新四类“识微/回顾/沉淀/洞察”对齐；历史页缺少日期筛选，查找会话困难。
8. **回顾页双击/长按交互丢失**：v2 重构后回顾卡片的双击编辑和长按菜单失效，需恢复。
9. **随机漫步视觉与操作冗余**：当前仅 3 张卡片、计数与灯泡图标干扰阅读、“已阅”功能与底部按钮布局不合理，需整体收紧。

---

## Solution

1. **内容智能折叠**：纯文字超过 12 行时折叠为省略态，多媒体单独缩略为一行；点击展开/收起；双击编辑、长按菜单在移动端恢复。
2. **语言入口平铺**：取消“界面语言”独立模块，把“中文 / English”平铺到“对话模型”上方。
3. **抽屉毛玻璃化**：设置抽屉后方遮罩改为约 70% 透明度毛玻璃，保留后方应用版面可见。
4. **标签区收纳化**：“所有标签”支持点击展开/收起；“管理标签 >”改为设置图标。
5. **标签快捷菜单**：每个标签提供展开/收起按钮和三个点“更多”菜单，菜单内置顶、编辑名称和图标、仅移除标签、删除标签和笔记。
6. **日历统计对齐合并口径**：弹窗中间统计项由“日记”改为“回顾”，数量按“旧日记 + 旧回顾”合并计算。
7. **Copilot 选择器与历史筛选**：RAG 选择器文案改为“识微 / 回顾 / 沉淀 / 洞察”，检索索引同步映射；历史页增加日期选择器，复用 RAG 日期样式。
8. **回顾交互恢复**：回顾内容卡片恢复双击进入 inline 编辑、长按弹出选项菜单。
9. **随机漫步精简**：卡片数改为 7 张；移除左上角计数、灯泡图标与“已阅”按钮；设置图标移到原灯泡位；降低后方卡片透明度、卡片宽度缩小 1.2 倍；剩余 5 个底部按钮随卡片尺寸重新排版。

## User Stories

### 内容折叠与交互恢复
1. 作为用户，我希望拾微/回顾内容文字超过 12 行后自动折叠，以免一屏被单条记录占满。
2. 作为用户，我希望折叠后点击一下展开、再点击一下收起，以便快速浏览长内容。
3. 作为用户，我希望 12 行统计不包含图片/视频/音频等多媒体占位，以便纯文本阅读节奏可控。
4. 作为用户，我希望超过 12 行时多媒体只展示一行缩略，不展开占用版面。
5. 作为用户，我希望在手机上仍能双击内容进入编辑，以便延续桌面习惯。
6. 作为用户，我希望在手机上长按内容弹出操作菜单，以便快速复制/编辑/删除。
7. 作为用户，我希望不超过 12 行时文字和多媒体正常完整展示，不影响短内容阅读。

### 设置页语言入口
8. 作为用户，我希望“界面语言”不要单独一个模块，以便设置页更紧凑。
9. 作为用户，我希望“中文 / English”直接放在“对话模型”上方，以便一眼找到并切换。

### 设置抽屉遮罩
10. 作为用户，我希望打开设置抽屉时后方应用版面仍隐约可见，以便感知自己没有离开当前页面。
11. 作为用户，我希望抽屉后方是毛玻璃效果而非灰色实底，以便视觉更通透。

### 抽屉标签区
12. 作为用户，我希望“所有标签”默认可收起，点击展开、再点击收起，以便抽屉更整洁。
13. 作为用户，我希望“管理标签”入口用图标表示而非“管理标签”四个汉字，以便更简洁。

### 标签快捷操作
14. 作为用户，我希望每个标签右侧有三个点菜单，点击可置顶该标签，以便高频标签排在前面。
15. 作为用户，我希望标签菜单支持“编辑名称和图标”，以便随时调整标签。
16. 作为用户，我希望标签菜单支持“仅移除标签”，以便保留笔记但清理标签分类。
17. 作为用户，我希望标签菜单支持“删除标签和笔记”，以便彻底清理某类内容。
18. 作为用户，我希望危险操作（移除/删除）用警示色区分，以免误触。

### 日历弹窗统计
19. 作为用户，我希望日历弹窗中间显示“回顾”而非“日记”，以匹配当前数据模型。
20. 作为用户，我希望“回顾”数量按合并后的日记+回顾计算，以便看到真实的回顾总量。

### Copilot 选择器与历史筛选
21. 作为用户，我希望 Copilot RAG 选择器显示“识微 / 回顾 / 沉淀 / 洞察”，与四大板块语义一致。
22. 作为用户，我希望选择“识微”时只检索拾微记录，选择“回顾”时检索合并后的回顾数据，选择“沉淀”时检索沉淀笔记，选择“洞察”时检索洞察内容。
23. 作为用户，我希望 Copilot 历史页顶部也有日期筛选器，以便按日期定位历史会话。
24. 作为用户，我希望历史页的日期筛选器样式与 RAG 页一致，以降低学习成本。

### 回顾页交互恢复
25. 作为用户，我希望双击回顾卡片直接进入 inline 编辑，以便快速修改内容。
26. 作为用户，我希望长按回顾卡片弹出复制/编辑/重新生成/删除等操作菜单。

### 随机漫步
27. 作为用户，我希望随机漫步一次展示 7 张卡片而非 3 张，以便一次浏览更多内容。
28. 作为用户，我希望移除卡片计数（如 1/3），以减少阅读干扰。
29. 作为用户，我希望移除顶部灯泡图标及其相关逻辑，使界面更干净。
30. 作为用户，我希望设置/筛选图标移到原灯泡位置，方便单手操作。
31. 作为用户，我希望后方卡片透明度降低，避免前后文字重影影响阅读。
32. 作为用户，我希望卡片宽度缩小 1.2 倍但高度不变，使一屏内信息更聚焦。
33. 作为用户，我希望移除底部“已阅”按钮，简化操作栏。
34. 作为用户，我希望移除“已阅”后剩余 5 个底部按钮的大小和排版随卡片尺寸自适应。

## Implementation Decisions

### 需求 1：内容折叠与移动端交互恢复
- **折叠阈值**：纯文本内容（按渲染行数）达到 12 行后进入折叠态；多媒体（图片、视频、音频、数字摘要块等）不计入 12 行。
- **折叠态展示**：文字截断并显示省略；多媒体仅保留一行缩略展示（超出部分用 +N 或滚动处理，具体样式与现有 `MultimediaAttachments` 缩略逻辑一致）。
- **展开/收起**：单击折叠态区域展开，单击已展开区域收起；与双击编辑通过事件互斥（延迟单击或双击取消延迟）。
- **双击编辑**：拾微页复用现有 `Record.tsx` 的 `onDoubleClick` + `handleOpenEditModal`；回顾页复用 `Review.tsx` 的 `onDoubleClick` + inline 编辑。
- **长按菜单**：移动端恢复 `onTouchStart` 500ms 定时器 + `onTouchEnd`/`onTouchMove` 清除；桌面端保留 `onContextMenu`。
- **适用范围**：拾微页内容卡片、回顾页内容卡片；沉淀/洞察如已存在类似折叠逻辑则保持一致。
- **i18n**：复用现有 `review.expand` / `review.collapse` / `thoughts.expand` / `thoughts.collapse`，不新增文案 key。
- **跨需求约束**：折叠/展开的单击事件不得与附件区单击（打开灯箱/详情）冲突；双击编辑仍跳过附件区。

### 需求 2：设置页语言入口平铺
- 移除现有独立的“界面语言”section（`Settings.tsx` 中 `data-testid="language-section"` 区域）。
- 在“对话模型”模块上方、全屏详情页内容区起始位置，平铺一个横向胶囊切换器：中文 / English。
- 仅保留两个选项，沿用 `settings.languageZh` / `settings.languageEn` i18n key。
- 切换逻辑复用 `settingsStore.setLanguage`，提示文案 `settings.languageHint` 保留。
- 抽屉模式与全屏详情模式都显示同一语言入口（放在详情页全局顶部，不随 tab 变化）。

### 需求 3：设置抽屉毛玻璃化
- 抽屉遮罩由 `bg-black/40 backdrop-blur-sm` 改为更高透明度 + 毛玻璃（约 70% 可见后方内容），例如 `bg-white/30 backdrop-blur-md` 或等效效果，确保后方应用版面隐约可见。
- 抽屉本体（`motion.aside`）背景保持当前浅色，不改为透明；仅改遮罩层。
- 点击遮罩关闭抽屉的行为保留。
- 移动端红线：抽屉与遮罩层仍使用局部滚动/事件，不依赖 body 滚动。

### 需求 4：抽屉“所有标签”区展开收起与管理入口
- “所有标签”区块增加折叠态：默认展开或默认收起由当前行为决定（当前已展开），改为可点击标题行展开/收起。
- 标题行右侧“管理标签 >”改为设置图标（`Settings2` / `SlidersHorizontal` / 等语义图标），点击仍跳转“标签设置”全屏详情页。
- 收起时只显示标题行与图标；展开时显示标签列表。
- 标签列表滚动行为保留：仅区块内部滚动。

### 需求 5：标签快捷操作菜单
- 标签列表中的每一项右侧显示两个图标按钮：展开/收起子标签的向下箭头（ChevronDown/ChevronUp），以及三个点“更多”菜单（`MoreVertical`）。
- “更多”菜单为弹出式动作菜单（参考图 7），包含：
  - 置顶
  - 编辑名称和图标
  - 仅移除标签（红色/玫瑰色警示）
  - 删除标签和笔记（红色/玫瑰色警示）
- 点击菜单外部或滚动时关闭菜单。
- “置顶”：调整标签排序权重，置顶标签排在列表最前；持久化到 `db.tags`（如新增 `pinned` 布尔字段或 `sort_order` 数值）。
- “编辑名称和图标”：弹出编辑浮层，支持修改标签路径与图标（若当前 schema 无图标字段，则先仅支持改名，图标为后续预留）。
- “仅移除标签”：从所有关联记录中移除该标签，不删除记录本身；需确认弹窗。
- “删除标签和笔记”：删除标签定义，并级联删除所有带该标签的记录（拾微/回顾/沉淀/洞察，按标签关联范围）；需二次确认。
- i18n 新增 key：`tags.pinTop`、`tags.editNameAndIcon`、`tags.removeTagOnly`、`tags.deleteTagAndNotes`。
- 若一次实现风险高，可分阶段：第一阶段仅实现菜单 UI 与“置顶/编辑/仅移除”；“删除标签和笔记”可复用现有 `TagManagement.tsx` 删除逻辑。

### 需求 6：日历弹窗统计文案与口径
- `CalendarHeatmap.tsx` 中中间统计项标签由 `calendarHeatmap.diary`（当前渲染为“日记”）改为 `calendarHeatmap.review`（“回顾”）。
- 中间数量 `middleCount` 改为统计 `daily_reviews` 中 `entry_type='diary'` 与 `entry_type='review'` 的总和。
- 下方字数统计 `wordCountStats` 对应“回顾”时，合并计算 `ai_editorial`（旧日记）与 `ai_review`（旧回顾）的字数。
- 左侧“拾微”与右侧“天”保持不变。
- i18n 可复用现有 `calendarHeatmap.review`，无需新增 key。

### 需求 7：Copilot 选择器文案与历史页日期筛选
- `Copilot.tsx` 中 `MODULE_LABEL_KEY` 的显示文案由「记录/日记/回顾/洞察」改为「识微/回顾/沉淀/洞察」，内部代码字段保持英文标识以避免前后端类型断裂：
  - `record` → `copilot.moduleShiwu`（识微）
  - `review` → `copilot.moduleHuigu`（回顾）
  - `thoughts` → `copilot.moduleChendian`（沉淀）
  - `insight` → `copilot.moduleDongcha`（洞察）
- 对应 `modules` state 类型由 `Array<'record' | 'diary' | 'review' | 'insight'>` 调整为 `Array<'record' | 'review' | 'thoughts' | 'insight'>`；默认全选新四类。
- 索引/检索逻辑：
  - 识微（`record`） → `raw_logs`
  - 回顾（`review`） → `daily_reviews` 全表（含旧 `entry_type='diary'` 与 `'review'`）
  - 沉淀（`thoughts`） → `thoughts`
  - 洞察（`insight`） → `mingwu`
- 同步更新 `CopilotRetrievalFilters`、`CopilotCitation` 与 `retrieveCopilotContext` 中的模块联合类型，移除 `diary` 独立分支，将日记与回顾合并到 `review` 分支。
- i18n 新增 key：`copilot.moduleShiwu`、`copilot.moduleHuigu`、`copilot.moduleChendian`、`copilot.moduleDongcha`；原 `copilot.moduleDiary` 可保留或移除。
- 历史页（`navView === 'history'`）顶部增加一行筛选区：仅保留“全部日期”日期选择器（借用 RAG 页现有日期选择器组件与样式），不包含模块筛选。
- 历史列表按 `updated_at` 倒序；日期筛选后按选定范围过滤会话记录。

### 需求 8：回顾页双击/长按交互恢复
- 在 `Review.tsx` 回顾卡片上恢复 `onDoubleClick`：多选模式/生成中/编辑中不触发；双击进入 inline 编辑（`setEditingReviewId` + `setEditText`）。
- 恢复 `onTouchStart` 500ms 长按检测：长按弹出上下文菜单（复制/编辑/重新生成/删除）。
- 恢复 `onContextMenu` 桌面右键菜单。
- 与需求 1 的内容折叠协同：回顾卡片 header 单击控制展开/收起，内容区双击进入编辑；内容与折叠态共用同一套单击/双击互斥机制。
- 菜单内容与现有长按菜单一致：复制、编辑、重新生成、删除。

### 需求 9：随机漫步精简与视觉调整
- `RandomWalk.tsx` 中 `DRAW_COUNT` 由 3 改为 7。
- 移除顶部细栏中的 `1/N` 计数展示（保留灯泡？不，本次需求要求移除灯泡）。
- 移除顶部灯泡图标（`Lightbulb`）及其相关提示逻辑。
- 将设置图标（`Settings2`）从当前位置移到原灯泡位置（左上角），点击仍打开数据源/冷却期配置面板。
- 设置面板展开方向与位置随图标移动而调整。
- 卡片透明度：降低后方非当前卡片的透明度（例如由当前 `baimiao-card-bubble` 默认背景不透明度调整为更低，或在 Swiper EffectCards 样式中覆盖），避免前后文字重影。
- 卡片宽度缩小 1.2 倍：在保持父容器 padding 不变的前提下，卡片内部宽度或 Swiper 容器宽度除以 1.2；高度保持不变（仍 flex-1 填充可用空间）。
- 移除底部操作栏的“已阅”按钮（`walk-read`）。
- 剩余 5 个按钮（标签、编辑、复制、删除、换一批）调整 icon/文字大小与间距，使整体与缩小后的卡片宽度协调。
- “已阅”相关的 `handleRead`、`randomWalk.read` i18n、已阅状态过滤逻辑可保留在代码中供后续可能复用，但 UI 上不再展示；若确定永久移除，可清理相关 localStorage 状态。
- i18n：`randomWalk.read` 可保留但不再渲染；新增/复用无。

### 跨需求约束
- **移动端红线**：所有新交互（折叠展开、标签菜单、抽屉遮罩、随机漫步卡片）必须在局部滚动容器内完成，保持全局 `html/body overflow-hidden + overscroll-behavior-none`。
- **图标库**：设置/标签/Copilot/随机漫步用 `lucide-react`；TabBar 维持 `@phosphor-icons/react`。
- **数据兼容性**：Copilot 模块筛选字段重命名属于接口变更，需同步更新 `CopilotRetrievalFilters` 类型与 `retrieveCopilotContext` 实现；历史会话表 `copilot_conversations` schema 不变。
- **性能**：折叠态不要预渲染完整 Markdown，可用 `line-clamp-12` 或最大高度截断；展开后再完整渲染。
- **无障碍**：折叠/展开区域与三个点菜单保留可聚焦状态与 aria-label；危险操作在菜单中用文字颜色区分。

## Testing Decisions

### 测试原则
- 只测外部可观察行为（DOM 结构、用户交互后的可见结果、文案内容、数量计算），不测实现细节（CSS 类名、内部 state、私有函数）。
- 优先复用现有 E2E seam，不新建 seam；本次所有需求均可被现有 4 个 seam 覆盖。

### Seam 映射
- **`foundation-migration.test.ts`**：需求 2/3/4/5（设置页语言入口、抽屉毛玻璃、所有标签展开收起、标签更多菜单）、需求 6（日历统计文案与合并口径）、需求 7（Copilot 选择器文案与历史页日期筛选）。
- **`thoughts.test.ts`**：需求 1（12 行折叠展开、双击编辑、长按菜单）、需求 8（回顾双击编辑、长按菜单）。
- **`random-walk.test.ts`**：需求 9（7 张卡片、移除计数/灯泡/已阅、卡片尺寸与透明度、设置图标位置）。
- **`multimedia.test.ts`**：需求 1（折叠态下多媒体一行缩略、正常态完整展示）。

### 各需求测试重点
1. **内容折叠**：12 行阈值触发折叠、点击展开/收起、移动端双击编辑、长按菜单、多媒体不计入行数且折叠态仅一行缩略。
2. **语言入口**：“界面语言”独立模块不存在；“中文/English”平铺在“对话模型”上方；切换语言生效。
3. **抽屉毛玻璃**：遮罩不是纯黑/灰实底，后方应用版面可见；点击遮罩关闭抽屉。
4. **标签区展开收起**：“所有标签”点击展开/收起；“管理标签”入口为图标而非汉字。
5. **标签菜单**：每个标签有展开箭头和三个点菜单；菜单含置顶、编辑、仅移除标签、删除标签和笔记；危险项有警示色。
6. **日历统计**：弹窗中间显示“回顾”；数量等于旧日记+旧回顾总数；字数统计同样合并。
7. **Copilot 选择器**：RAG 选择器显示“识微/回顾/沉淀/洞察”；各选项过滤对应数据表；历史页顶部有日期筛选器。
8. **回顾交互**：双击回顾卡片进入 inline 编辑；长按/右键弹出操作菜单。
9. **随机漫步**：一次加载 7 张卡片；无 1/N 计数、无灯泡图标、无“已阅”按钮；设置图标位于左上角；后方卡片透明度降低、卡片宽度缩小；底部 5 按钮排版协调。

## Out of Scope

- 不改动数据同步、云备份、OAuth、端到端加密相关逻辑。
- 不涉及端侧 AI（Capacitor/LiteRT-LM）迁移（见 `docs/on-device-ai-migration.md`）。
- 不改动底层 AI 生成算法、Prompt 内容本身。
- 不新增 AI 模型服务商。
- 不改动日记/回顾/洞察/洞察的生成调度与任务队列逻辑。
- 不改动附件存储 schema（仅读取已有字段）。
- 不改底部 TabBar 名称（TabBar 仍显示“拾微/回顾/沉淀/洞察”；Copilot 选择器内使用“识微”是文案层面的区分）。
- 不改动标签的存储 schema 核心字段（如需支持图标/排序，可扩展但不影响现有查询）。

## Further Notes

- **命名说明**：需求 7 中 Copilot 选择器使用“识微”对应数据表 `raw_logs`，与 TabBar 文案“拾微”有所区别；PRD 按用户指定文案记录，实现时通过新增 i18n key 区分，不影响 TabBar。
- **“日记”数据合并**：V2 已将旧日记合并进 `daily_reviews.entry_type='diary'`，需求 6 的“回顾”统计口径即“全部 `daily_reviews` 记录”，实现时直接统计整个表即可。
- **随机漫步“已阅”**：当前代码中“已阅”会永久标记记录不再出现；本次需求要求 UI 移除该按钮。若希望保留“不再出现”的隐性逻辑，可改用其他方式（如全部展示后进入空状态）；若彻底移除，可清理 `random-walk-shown` 中 `read` 标记逻辑。具体实现优先级由开发时根据代码量决定。
- **标签菜单实现顺序**：建议先做“置顶”与“仅移除标签”（风险低），再做“编辑名称和图标”与“删除标签和笔记”（涉及数据级联）。
- **折叠行数计算**：12 行按渲染行数计算；由于中文排版存在换行，可用 `line-clamp-12` 作为近似实现，必要时用 ResizeObserver 精确测量； multimedia 不计入行数的规则通过在折叠态单独渲染多媒体缩略行来满足。
