# 需求 9：设置页展示方式重构

## Problem Statement

当前系统设置是左侧固定菜单 + 右侧内容区的分栏布局（Seam 2 实施），不够紧凑，移动端体验欠佳。用户希望改为更紧凑的移动端常见模式：左侧抽屉展示菜单和标签，点击某项后进入全页详情，顶部用横向导航栏切换不同设置项。

（PRD Problem Statement 第 9 条：系统设置左侧固定菜单 + 右侧分栏不够紧凑，移动端体验欠佳。）

## Solution

设置页重构：抽屉 + 全页详情 + 顶部横向导航栏（推翻 Seam 2 的左侧分栏）。只改展示方式，不改设置内容，实现抽屉菜单 + 全页设置详情 + 顶部横向导航栏的新交互。本需求推翻 Seam 2 的左侧分栏布局，设置页需重做。

（PRD Solution 第 9 条：设置页重构--抽屉 + 全页详情 + 顶部横向导航栏（推翻 Seam 2 的左侧分栏）。）

## User Stories

43. 作为用户，我希望点 ≡ 从左侧滑出抽屉（含设置菜单项 + 所有标签区块），以便统一查看。
44. 作为用户，我希望点抽屉菜单项进全屏设置详情页（顶部横向导航栏），以便专注配置。
45. 作为用户，我希望横向导航栏切换设置项（动画过渡 + 左右滑动），以便快速切换。
46. 作为用户，我希望横向导航栏用胶囊高亮当前项，以便明确位置。
47. 作为用户，我希望桌面端也用抽屉 + 全页模式（不保留左右分栏），以便跨设备一致。
48. 作为用户，我希望标签设置作为横向导航 tab + 抽屉"所有标签"快捷入口，以便便捷访问。
49. 作为用户，我希望标签多时仅"所有标签"区块内部滚动，抽屉其他部分不动，以便操作稳定。
50. 作为用户，我希望全屏详情页直接关闭设置（不返回抽屉），以便快速退出。

## Implementation Decisions

> 来源：PRD Implementation Decisions「需求 9」章节 + 需求基线「需求 9」已确认细节 + 相关跨需求约束。

### 左侧抽屉
- 点击顶部标题栏 `≡` 设置按钮后，从左侧滑出抽屉。
- 抽屉上半部分为原左侧设置菜单项（对话模型 / 语音朗读 / 向量与语义 / 数据管理 / 提示词配置 / 标签设置 / 关于）。
- 抽屉下半部分新增「所有标签」区块，展示现有标签列表。
- 「所有标签」区块右上角增加快捷入口，点击直接跳转「标签设置」。

### 点击菜单项跳转全页
- 点击抽屉中某个设置项，不再在右侧展开，而是跳转到全屏设置详情页。
- 全屏详情页顶部增加横向滚动导航栏，包含抽屉里所有设置项；点击横向导航栏项目切换详情。
- 抽屉设置项的图标、排序、分组与现在完全一致，不调整。

### 标签设置双入口
- 「标签设置」作为顶部横向导航栏一个 tab。
- 同时保留抽屉「所有标签」右上角快捷入口。

### 桌面端适配
- 桌面端宽屏也采用抽屉 + 全页模式，不保留原左右分栏。

### 全屏详情页关闭
- 全屏详情页直接关闭设置，不再返回抽屉。

### 横向导航栏交互
- 点击 tab 切换有动画过渡。
- 支持左右滑动切换 tab。
- 胶囊/药丸样式高亮当前选中 tab。

### 标签区块滚动
- 标签数量多时仅「所有标签」区块内部滚动，抽屉其他部分不跟随。
- 区块高度根据整体排版自适应，不强制固定比例。

### 相关跨需求约束
- **需求 9 推翻 Seam 2**：之前 Seam 2 实施的"左侧固定菜单 + 右侧内容分栏"被需求 9 推翻，改抽屉 + 全页 + 横向导航。设置页需重做。
- **G3 冲突处理**：标签管理在设置抽屉（本需求），顶部栏移除的是标签图标（TagsIcon）非随机漫步；随机漫步保留 header 灯泡入口（不迁设置）。
- **移动端红线**：全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`；设置抽屉/全屏详情页/标签区块作为新滚动区用局部 `overflow-y-auto`，不依赖 body 滚动。
- **图标库**：设置用 lucide-react。
- **数据兼容性**：Settings store 版本 v11 -> v12，在同一 migrate 函数中协调 Prompt 合并 + TTS 外部配置字段初始化（本需求只改展示方式，不改 store 结构，但需与 migrate 协调不冲突）。

## Testing Decisions

- seam: `foundation-migration.test.ts`（需求 9：设置页抽屉 + 全页 + 横向导航）
- 原则：只测外部可观察行为（UI 渲染/交互/可见结果），不测实现细节（CSS 类名/内部状态）
- 测试重点（PRD Testing 第 9 条）：设置页--抽屉滑出、菜单 + 标签区块、点项跳全页、横向导航切换、胶囊高亮、桌面同模式、标签滚动。即：
  - 点击 `≡` 从左侧滑出抽屉。
  - 抽屉含设置菜单项 + 「所有标签」区块。
  - 点击抽屉菜单项跳转全屏设置详情页（不再右侧展开）。
  - 横向导航栏点击切换设置项（动画过渡 + 左右滑动 + 胶囊高亮当前）。
  - 桌面端宽屏同样用抽屉 + 全页模式（不保留左右分栏）。
  - 标签数量多时仅「所有标签」区块内部滚动，抽屉其他部分不跟随。
  - 全屏详情页直接关闭设置（不返回抽屉）。
  - 标签设置作为横向导航 tab + 抽屉「所有标签」快捷入口双入口可达。

## Out of Scope

- 不改数据同步/云备份/OAuth
- 不涉端侧 AI 迁移
- 不改 AI 生成算法
- 不改设置内容（只改展示方式：各设置项的功能逻辑、配置项、字段、存储结构不变）
- 不改各设置项的实际业务逻辑（对话模型/语音朗读/向量与语义/数据管理/提示词配置/标签设置/关于的配置行为不变）
- 不改附件存储 schema（仅读取已有字段）
- 不改顶部栏 `≡` 按钮本身的位置/样式（≡ 入口由需求 6 顶部标题栏重构定义；本需求只负责 ≡ 点击后滑出的抽屉及后续设置页展示）
