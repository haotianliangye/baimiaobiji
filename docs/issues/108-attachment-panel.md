# 需求 8：拾微附件上传入口动画面板

## Problem Statement

拾微页底部输入框附件上传按钮（回形针）点击后直接调起系统文件选择器，缺中间面板选上传类型。参考微信聊天输入，点击 + 号后应从底部滑出功能面板供用户选择上传类型。

（PRD Problem Statement 第 8 条：拾微附件上传直接调系统文件选择器，缺中间面板选上传类型。）

## Solution

拾微附件上传动画面板：点击附件按钮从底部上滑网格面板，6 选项。点击拾微输入框旁附件按钮后，以动画展开底部附件选择面板，面板内列出可用上传类型。

（PRD Solution 第 8 条：拾微附件上传动画面板--点击附件按钮从底部上滑网格面板，6 选项。）

## User Stories

39. 作为用户，我希望点附件按钮从底部上滑面板选类型，以便直观选择。
40. 作为用户，我希望面板有相册/音频/视频/链接/文件/取消 6 项大图标，以便覆盖所有类型。
41. 作为用户，我希望点取消/面板外部/下滑都能关闭面板，以便灵活退出。
42. 作为用户，我希望面板上滑时把输入栏和主内容向上推，以便层级清晰。

## Implementation Decisions

> 来源：PRD Implementation Decisions「需求 8」章节 + 需求基线「需求 8」已确认细节 + 相关跨需求约束。

### 触发入口
- 拾微页底部输入框旁回形针/附件按钮。
- 仅拾微页使用。

### 面板形态
- 点附件按钮后从底部上滑面板（参考微信聊天输入底部面板），网格排列功能图标。
- 面板从底部滑出，把输入栏和上方主内容一起向上推，层级清晰。
- 动画时长和缓动曲线按系统默认。

### 面板内容
- 6 项：相册 / 音频 / 视频 / 链接 / 文件 / 取消。
- 图标用 lucide-react（G8 补充）：
  - 相册 `Image`
  - 音频 `Music`
  - 视频 `Video`
  - 链接 `Link`
  - 文件 `FileUp`
- 面板内容沿用现有选项。

### 关闭方式
- 三种并存：点击「取消」/ 点击面板外部 / 下滑手势。

### 相关跨需求约束
- **移动端红线**：全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`；附件面板作为新滚动区用局部 `overflow-y-auto`，不依赖 body 滚动。
- **无障碍**：附件面板选项需有清晰焦点状态。
- **图标库**：面板用 lucide-react。
- **附件存储**：本需求仅改入口面板（选类型），不改各附件类型的实际上传/存储逻辑，不改附件存储 schema。

## Testing Decisions

- seam: `multimedia.test.ts`（需求 8：附件上传面板 6 选项 + 关闭方式）
- 原则：只测外部可观察行为（UI 渲染/交互/可见结果），不测实现细节（CSS 类名/内部状态）
- 测试重点（PRD Testing 第 8 条）：附件面板--底部上滑、6 选项、三方式关闭、上推主内容。即：
  - 点击附件按钮后从底部上滑出面板。
  - 面板含 6 项：相册/音频/视频/链接/文件/取消。
  - 三方式关闭：点取消 / 点面板外部 / 下滑手势。
  - 面板上滑时输入栏和主内容被向上推。
  - 仅拾微页使用。

## Out of Scope

- 不改数据同步/云备份/OAuth
- 不涉端侧 AI 迁移
- 不改 AI 生成算法
- 不改附件存储 schema（仅读取已有字段）
- 不改各附件类型（相册/音频/视频/链接/文件）的实际上传/存储/摘要逻辑（本需求仅改入口选类型面板）
- 不涉其他页面（仅拾微页使用该面板）
