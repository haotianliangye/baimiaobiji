# 需求 4：碎屑/回顾卡片双击编辑

## Problem Statement

编辑碎屑和回顾都需长按/右键菜单或展开后点编辑按钮，操作路径长。用户希望像桌面端习惯，双击卡片直接进入编辑，缩短操作链路。

## Solution

给碎屑页和回顾页内容卡片增加双击编辑能力：双击卡片直接进编辑（碎屑弹窗、回顾 inline）。碎屑页双击打开需求 3 的 RichEditor 编辑弹窗，回顾页双击触发现有 inline 编辑（展开后 textarea 获得焦点）。多选模式下禁用双击编辑以免误操作。

## User Stories

18. 作为用户，我希望双击碎屑卡片直接打开编辑弹窗，以便快速编辑。
19. 作为用户，我希望双击回顾卡片触发 inline 编辑（textarea 获得焦点），以便快速编辑。
20. 作为用户，我希望多选模式下双击不触发编辑，以免误操作。
21. 作为用户，我希望碎屑附件区单击仍打开灯箱/详情，双击只在非附件区触发编辑，以便两个行为不冲突。

## Implementation Decisions

### 碎屑页双击编辑
- `Record.tsx` 的 `LogCard` 绑定 `onDoubleClick`（卡片任意区域除操作按钮），双击打开需求 3 的 RichEditor 弹窗，弹窗内所有字段可编辑。
- 附件区单击保留打开灯箱/详情，双击只在非附件元素触发编辑，避免两个行为冲突。
- 碎屑卡片单击保持现有单击行为不变。

### 回顾页双击编辑
- `Review.tsx` 回顾内容卡片绑定 `onDoubleClick`，双击触发现有 inline 编辑（展开后 textarea 获得焦点）。
- 维持现有 inline 编辑机制（卡片展开后内部变 textarea），双击仅作为触发该 inline 编辑状态的入口。
- 回顾卡片单击保持折叠/展开行为不变。

### 单击/双击互斥
- 用 `onDoubleClick` 事件实现，不引入双击库。
- 处理单击与双击互斥：回顾页单击折叠/展开，双击进编辑。
- 碎屑页单击保持现有行为，双击进编辑。

### 多选模式
- 多选模式不响应双击编辑。
- 多选状态由父组件统一控制，进入多选后移除双击监听。

### 入口联动
- 本需求为碎屑编辑弹窗提供双击入口，弹窗本身实现由需求 3 负责。
- 保留长按/右键菜单"编辑记录"入口，双击为新增入口。

### 跨需求约束（相关）
- **多媒体卡片细节（G9/G10）**：附件区单击打开灯箱/详情的行为需保留，双击编辑仅在非附件区触发，与混合媒体布局（图片/视频 2×2 网格 -> 音频纵向列表 -> AI 摘要文本）共存不冲突。
- **移动端红线**：全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`；本需求不新增滚动区。

## Testing Decisions

- seam: `thoughts.test.ts`
- 原则：只测外部可观察行为（UI 渲染/交互/可见结果），不测实现细节（CSS 类名/内部状态）
- 测试重点：
  - 碎屑卡片双击打开编辑弹窗
  - 回顾卡片双击触发 inline 编辑（textarea 获得焦点）
  - 多选模式下双击不触发编辑
  - 碎屑附件区单击打开灯箱/详情，双击仅在非附件区触发编辑（两行为不冲突）

## Out of Scope

- 不改数据同步/云备份/OAuth
- 不涉端侧 AI 迁移
- 不改 AI 生成算法
- 不改碎屑编辑弹窗本身的实现（RichEditor 弹窗由需求 3 负责），本需求仅提供双击入口
- 不改回顾 inline 编辑的内部实现（复用现有），本需求仅提供双击触发
- 不引入双击库（用原生 `onDoubleClick` 事件实现）
- 不改单击行为（碎屑单击保持现有、回顾单击保持折叠/展开）
- 不改附件存储 schema（仅读取已有字段）
