# Seam 5：沉淀卡片限高折叠与缩略图规范

## Problem Statement

沉淀页长文卡片无限延伸，单张卡片即可占满屏幕；卡片内多媒体缩略图比例与数量不统一，影响浏览效率。

## Solution

为沉淀卡片引入折叠态与展开态，并规范缩略图展示：
- 时间线视图最多 7 行，瀑布流视图最多 12 行，超出折叠并显示“展开”按钮。
- 单击卡片展开/收起，双击进入编辑。
- 卡片内多媒体缩略图统一 1:1 比例：时间线一行最多 3 个，瀑布流一行最多 2 个。

## User Stories

1. 作为用户，我希望沉淀卡片在长文时不会无限延伸，而是折叠为限高框，以便一屏能看到多条记录。
2. 作为用户，我希望时间线视图下沉淀卡片最多显示 7 行，瀑布流视图最多 12 行，超出显示“展开”按钮，以便在不同视图下都有合适的预览高度。
3. 作为用户，我希望单击沉淀卡片展开/收起，双击进入编辑，以便兼顾浏览与编辑效率。
4. 作为用户，我希望沉淀卡片的图片/音频/视频缩略图在时间线视图下一行最多 3 个、瀑布流视图下一行最多 2 个，统一 1:1 比例，以便在窄卡片中也能看清。

## Implementation Decisions

- 沉淀卡片（时间线与瀑布流两种视图）需要支持折叠态与展开态。
- 折叠态最大高度：时间线 7 行，瀑布流 12 行（按正文行高计算）。
- 超出部分使用渐变遮罩 + “展开”按钮提示。
- 单击卡片切换展开/折叠；双击进入编辑。
- 单击/双击判别：约 300ms 内第二次点击视为双击，取消单击动作。
- 卡片内多媒体缩略图统一 1:1 比例：
  - 时间线：一行最多 3 个。
  - 瀑布流：一行最多 2 个。
  - 超出显示 “+N” 或进入详情。

## Testing Decisions

- 验证长文卡片折叠到指定高度并显示“展开”。
- 验证单击展开/收起，双击进入编辑（不触发展开）。
- 验证时间线/瀑布流缩略图一行 3/2 个，1:1 比例。
- 验证 300ms 内双击与单击的判别逻辑。
- 验证移动端长按/触摸与桌面端鼠标行为一致。

## Out of Scope

- 不改动沉淀内容的生成与编辑逻辑本身。
- 不改动瀑布流/时间线视图切换机制。
- 顶部栏统一规则见 Seam 1。
