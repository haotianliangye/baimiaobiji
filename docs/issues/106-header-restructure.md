# 需求 6：顶部标题栏与次级导航重构

## Problem Statement

四个主 Tab（碎屑/回顾/沉思/明悟）各有独立次级导航栏/页面内 header，顶部区域堆叠、空间浪费。沉思/明悟中间区域缺下拉胶囊切换器。碎屑/回顾页点击日期弹日历的功能分散在页面内，需迁移到顶部标题栏统一收敛。

（PRD Problem Statement 第 6 条：四个主 Tab 各有独立次级导航栏，顶部堆叠；沉思/明悟中间区域缺下拉胶囊切换器。）

## Solution

重构顶部标题栏，移除各页面次级导航栏，把日期选择、视图切换、时间范围选择收敛到标题栏内。统一四 Tab 顶部栏结构（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡）：碎屑/回顾中间为日期选择器，沉思为瀑布流/时间线下拉胶囊切换器，明悟为时间范围下拉胶囊。

（PRD Solution 第 6 条：顶部标题栏重构——统一四 Tab 顶部栏（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡），沉思下拉胶囊切换器、明悟时间范围下拉胶囊。）

## User Stories

25. 作为用户，我希望四个主 Tab 顶部栏统一结构（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡），以便一致体验。
26. 作为用户，我希望标题文字不可点击，仅 ≡ 图标打开系统设置，以免误触。
27. 作为用户，我希望碎屑/回顾中间是日期选择器（月-日 + 左右箭头 + 点击弹选择器），以便切日期。
28. 作为用户，我希望沉思中间是瀑布流/时间线下拉胶囊切换器，以便切视图。
29. 作为用户，我希望明悟中间是时间范围下拉胶囊（今日/本周/本月/本季度/半年/一年/自选），以便选范围。
30. 作为用户，我希望日期字号比标题略小，以便视觉层次。
31. 作为用户，我希望沉思胶囊文案是"瀑布流/时间线"仅文字无图标，以便简洁。

## Implementation Decisions

> 来源：PRD Implementation Decisions「需求 6」章节 + 需求基线「需求 6」已确认细节 + 相关跨需求约束。

### 涉及范围
- 四个主 Tab（碎屑/回顾/沉思/明悟）。
- 移除各页面次级导航栏，内容合并到统一顶部标题栏。
- 碎屑/回顾页点击日期弹日历的功能迁移到顶部标题栏日期选择器。

### 左侧区域
- `[≡] {当前页面标题}`。
- 标题随 Tab 变化：碎屑=**白描**、回顾=**回顾**、沉思=**沉思**、明悟=**明悟**。
- 仅点击 `≡` 打开系统设置；标题文字不再响应点击。

### 中间区域
- **碎屑/回顾**：居中日期选择器 `< 日期 >`。
  - 日期只显示**月-日**。
  - 左右箭头保留切换上一天/下一天。
  - 点击日期文本弹出选择器。
  - **日期字号比标题略小**（gaps G4）。
- **沉思**：中间区域为瀑布流/时间线**下拉式胶囊切换器**，点击切换视图（gaps G1）。文案 `瀑布流`/`时间线`，仅文字不显示图标。
- **明悟**：中间区域为时间范围**下拉胶囊**（如"本周"），点击展开选择（gaps G2）。复用现有洞察下拉菜单选项与行为；选项：今日、本周、本月、本季度、半年、一年、自选范围；弹窗内排版横排网格**每行 4 个，共两行**（第一行 4、第二行 3）。

### 右侧区域
- 保持现有搜索、RAG、随机漫步灯泡三个图标不变。

### 日期选择器显示范围
- 仅碎屑/回顾显示日期选择器；沉思/明悟中间由各自胶囊替代。

### 标题栏高度
- 维持现有高度不变。

### 相关跨需求约束
- **移动端红线**：全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`；日期选择器/胶囊弹窗等新滚动区用局部 `overflow-y-auto`，不依赖 body 滚动。
- **无障碍**：顶部栏按钮保留 `aria-label`。
- **图标库**：顶部栏/设置/面板用 lucide-react；底部 TabBar 用 @phosphor-icons/react。
- **G3 冲突处理**：随机漫步保留 header 灯泡入口（不迁设置）；顶部栏移除的是**标签图标**（TagsIcon）非随机漫步；标签管理在设置抽屉（需求 9）。即本需求在收敛次级导航时，移除顶部栏的标签图标，右侧仅留搜索·RAG·灯泡三图标。

## Testing Decisions

- seam: `foundation-migration.test.ts`（顶部栏结构：≡ 标题/日期/胶囊/搜索·RAG·灯泡）
- 原则：只测外部可观察行为（UI 渲染/交互/可见结果），不测实现细节（CSS 类名/内部状态）
- 测试重点（PRD Testing 第 6 条）：顶部栏——四 Tab 统一结构、标题不可点击、日期选择器、沉思胶囊、明悟胶囊、日期字号。即：
  - 四个主 Tab 顶部栏统一结构（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡）。
  - 标题文字不可点击，仅 ≡ 图标打开系统设置。
  - 碎屑/回顾中间日期选择器（月-日 + 左右箭头 + 点击弹选择器），日期字号比标题略小。
  - 沉思中间瀑布流/时间线下拉胶囊切换器（文案"瀑布流/时间线"仅文字无图标）。
  - 明悟中间时间范围下拉胶囊（今日/本周/本月/本季度/半年/一年/自选范围）。
  - 各页面次级导航栏已移除、内容合并到顶部标题栏。

## Out of Scope

- 不改数据同步/云备份/OAuth
- 不涉端侧 AI 迁移
- 不改 AI 生成算法
- 不实现系统设置抽屉/全页详情/横向导航栏本身（≡ 按钮仅作为入口，设置页展示方式重构由需求 9 负责）
- 不改随机漫步灯泡入口行为（灯泡图标保留，随机漫步主内容区模式由需求 2 负责）
- 不改搜索/RAG 图标的现有行为（仅保留位置不变）
- 不改日期存储/查询逻辑（日期选择器仅迁移 UI 入口，不改 `?date=`/`review_date` 数据流）
- 不改沉思瀑布流/时间线视图的内部渲染实现（仅提供胶囊切换器 UI，视图本身不变）
- 不改明悟洞察生成的现有选项语义（时间范围胶囊复用现有洞察下拉选项与行为）
- 不改附件存储 schema
