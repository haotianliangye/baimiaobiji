# 白描笔记 UI/UX 重构 PRD

> 状态：**ready-for-agent**（已 seams 确认，可在本地按 9 个 seam 拆分实施）。  
> 对齐来源：`docs/requirement-alignment-2026-07-13.md`  
> PRD 文件：`docs/prd-ui-ux-2026-07-13.md`

---

## Problem Statement

白描笔记当前的主界面、系统设置、RAG+CHAT 板块、多媒体记录展示、沉淀卡片、附件上传面板等 UI/UX 存在多处不一致和体验断层：

- 多媒体记录（图片/视频/音频）在拾微列表中缺少缩略图与 AI 摘要的正确展示，用户无法直观判断附件内容。
- 顶部导航层级混乱：存在“白描笔记”标题、次级导航栏、设置图标、标签图标、随机漫步图标、Copilot 入口等多个入口，视觉重心分散，且“白描”点击触发设置弹窗容易误触。
- 系统设置页采用横向标签，模块一多即拥挤；语言切换按钮排版不统一；“关于”信息以弹窗形式存在，与设置页割裂。
- RAG+CHAT（Copilot）板块使用“对话 / 历史会话”二级标签 + “RAG 问答 / 通用 Chat”模式切换，导航层级冗余。
- 沉淀页存在独立的“瀑布流 / 时间线”切换区和灯泡入口，与统一顶部栏规则冲突；长文卡片无限延伸，单张卡片即可占满屏幕。
- 拾微页附件上传入口仅为图片图标，无法表达已支持的图片/音频/视频/文件/链接等能力。
- 提示词配置中“明悟 / 洞察 / 日记 / 回顾”的摘要与生成 prompt 存在重复和语义混淆。
- TTS 仅支持浏览器内置，无法接入 Gemini / 火山引擎等外部 API。
- 卡片右键/长按菜单出现 `view.copyContent编辑内容` 等中英混编文案。
- 底部 TabBar 的“沉淀 / 明悟”图标与页面语义不匹配。

用户希望在一轮统一重构中解决上述问题，使各板块导航、设置、展示、输入体验达成一致，并为后续功能扩展留下清晰的结构。

---

## Solution

对全站 UI/UX 进行一轮以“统一顶部栏、重构系统设置、规范多媒体展示、优化输入与导航”为主线的重构：

1. **统一顶部标题栏**：四个主 Tab（拾微、回顾、沉淀、明悟）共用同一套顶部栏规则——左侧 `[≡] {页面标题} · {副标题}`，中间日期，右侧固定 `搜索 → RAG+CHAT → 灯泡`。标题不再可点击。
2. **重构系统设置**：从横向标签改为左侧竖排菜单 + 右侧内容区（桌面端常驻，移动端 push 抽屉动画）；新增“关于”标签页，并入原关于弹窗内容；新增“语音朗读”和“标签设置”菜单项；语言切换改为统一一行的胶囊按钮。
3. **规范多媒体展示**：拾微卡片内图片/视频以 2×2 缩略图网格展示（16:9，超出 “+N”），音频以播放器控件展示，均附带 AI 摘要；摘要失败可单附件重试。
4. **沉淀卡片限高折叠**：时间线最多 7 行、瀑布流最多 12 行，超出折叠并显示“展开”按钮；单击展开/收起，双击进入编辑。
5. **沉淀缩略图 1:1 一行三列（时间线）/ 一行两列（瀑布流）**。
6. **输入工具栏升级**：沉淀富文本输入框工具栏增加通用上传、超链接、麦克风、标签 `#`、更多（表格/代码块等）。
7. **拾微附件上传面板**：点击附件按钮从底部上滑网格面板，提供相册、音频、视频、链接、文件、取消六个选项。
8. **RAG+CHAT 导航重构**：移除“对话 / 历史会话”二级标签，改为顶部一行 `RAG → CHAT → 历史` 导航。
9. **提示词配置合并**：明悟与洞察生成 Prompt 合并为“明悟和洞察生成 Prompt”；日记与回顾一句话摘要合并为“日记回顾一句话摘要生成 Prompt”；洞察摘要改为“明悟和洞察一句话摘要生成 Prompt”。
10. **TTS 外部 API**：支持 Gemini / 火山引擎外部 TTS，配置字段与对话模型一致。
11. **i18n 修正**：全量修正右键/长按菜单及按钮的中英混编文案。
12. **TabBar 图标替换**：沉淀使用原明悟图标，明悟使用 `sun-dim` 图标。

## User Stories

### 顶部导航与系统设置

1. 作为用户，我希望顶部标题栏在四个主 Tab 间保持一致，以便在不同板块间快速建立操作预期。
2. 作为用户，我希望只有设置图标（≡）能打开系统设置，而不是点击标题文字，以避免误触。
3. 作为用户，我希望四个主 Tab 的顶部栏右侧始终有搜索、RAG+CHAT 入口和灯泡，以便随时使用全局能力。
4. 作为用户，我希望拾微/回顾/沉淀页顶部显示“今日 X 字”，以便一眼看到当日输入量。
5. 作为用户，我希望日期居中并保留左右箭头切换，同时点击日期仍能打开日期选择器，以便快速跳转日期。
6. 作为用户，我希望“白描”替代“白描笔记”作为标题，并保留“白描”的简洁品牌感。
7. 作为用户，我希望系统设置采用左侧菜单 + 右侧内容区的布局，以便在模块增多时仍能清晰导航。
8. 作为用户，我希望系统设置在桌面端左侧菜单常驻、在移动端以 push 抽屉动画展开，以便在不同设备上都有良好体验。
9. 作为用户，我希望“关于”信息（版本、作者、检查更新、反馈）放在系统设置内的“关于”标签页，而不是单独弹窗，以便统一查找。
10. 作为用户，我希望语言切换按钮“中文 / English”排版统一为一行且高亮当前语言，以便界面更整齐。
11. 作为用户，我希望标签管理从顶部栏图标迁移到系统设置内的“标签设置”，以便顶部栏更简洁。

### RAG+CHAT 板块

12. 作为用户，我希望 Copilot 页面顶部只有一行 `RAG → CHAT → 历史` 导航，以便快速切换三种对话视图。
13. 作为用户，我希望 RAG 模式下原有的记录/日记/回顾/洞察筛选胶囊、日期筛选、日记模板筛选全部保留，以便继续按条件检索本地知识。
14. 作为用户，我希望从任意主 Tab 点击 RAG+CHAT 入口都能进入该板块，并看到统一的顶部栏规则。

### 多媒体展示

15. 作为用户，我希望拾微卡片中的图片和视频显示缩略图，而不是文字占位，以便快速识别内容。
16. 作为用户，我希望单张图片以 16:9 比例撑满卡片宽度，多张图片以 2×2 网格展示，超出显示 “+N”，以便平衡预览密度与可读性。
17. 作为用户，我希望视频缩略图叠加播放图标，点击可播放，以便区分视频与图片。
18. 作为用户，我希望音频以播放器控件展示，并显示 STT 转写摘要，以便了解音频内容。
19. 作为用户，我希望图片/视频的 AI 摘要显示在媒体区域下方，最多三行，点击进入详情可看完整摘要，以便快速浏览。
20. 作为用户，我希望当某个附件的 AI 摘要生成失败时，该附件右下角显示“重新生成”按钮，单独重试该附件，以免浪费 token。
21. 作为用户，我希望音频转写失败时，播放器旁显示“重新转写”按钮，以便单独重试。
22. 作为用户，我希望摘要正在生成时显示弱提示“AI 摘要生成中…”，以建立耐心等待的心智。

### 沉淀卡片

23. 作为用户，我希望沉淀卡片在长文时不会无限延伸，而是折叠为限高框，以便一屏能看到多条记录。
24. 作为用户，我希望时间线视图下沉淀卡片最多显示 7 行，瀑布流视图最多 12 行，超出显示“展开”按钮，以便在不同视图下都有合适的预览高度。
25. 作为用户，我希望单击沉淀卡片展开/收起，双击进入编辑，以便兼顾浏览与编辑效率。
26. 作为用户，我希望沉淀卡片的图片/音频/视频缩略图在时间线视图下一行最多 3 个、瀑布流视图下一行最多 2 个，统一 1:1 比例，以便在窄卡片中也能看清。

### 输入与附件

27. 作为用户，我希望沉淀页富文本输入框工具栏有通用上传按钮，支持图片/音频/视频，以便统一上传入口。
28. 作为用户，我希望工具栏有超链接按钮，点击后弹出输入框插入 Markdown 链接，以便快速加链接。
29. 作为用户，我希望工具栏有麦克风按钮，点击录音并转写后插入文本，以便语音输入。
30. 作为用户，我希望工具栏有 `#` 标签按钮，点击插入标签或弹出标签选择器，以便快速打标签。
31. 作为用户，我希望工具栏有“…”更多按钮，展开后包含表格、代码块、内联代码、导出、预览 Markdown，以便使用高级编辑功能。
32. 作为用户，我希望拾微页底部附件按钮点击后从底部上滑面板，面板内有相册、音频、视频、链接、文件、取消六个大图标选项，以便直观选择附件类型。
33. 作为用户，我希望点击面板外部或取消按钮都能关闭附件面板，以便灵活退出。

### 提示词与 TTS 设置

34. 作为用户，我希望明悟和洞察共用一套生成 Prompt 配置，减少重复设置。
35. 作为用户，我希望日记和回顾的一句话摘要合并为一个 Prompt，因为摘要逻辑本质相同。
36. 作为用户，我希望设置中“明悟和洞察生成 Prompt”的标签为明悟/洞察/自定义 1/2/3，并保留自动生成选中复选框，以便灵活控制哪些类型自动产出。
37. 作为用户，我希望 TTS 支持选择“浏览器内置”或“外部 TTS API”，选择外部 API 时填写 Provider / API Key / Base URL / Model / Voice / 默认语言 / 语速，以便使用 Gemini 或火山引擎朗读。

### 文案与图标

38. 作为中文用户，我希望右键/长按菜单显示纯中文，不出现 `view.copyContent编辑内容` 等混编文案。
39. 作为英文用户，我希望英文环境下所有菜单、按钮、Tooltip 显示纯英文。
40. 作为用户，我希望回顾页生成弹窗的标题从“生成 2 篇回顾”改为“生成 2 篇”，因为选中项可能包含日记或自定义类型。
41. 作为用户，我希望底部 TabBar 中“沉淀”使用更符合沉淀含义的图标，“明悟”使用太阳图标，以便图标与板块语义匹配。

## Implementation Decisions

### Seam 1：全局 Layout 与顶部标题栏

- 统一四个主 Tab 的顶部标题栏结构：左侧 `[≡] {页面标题} · {副标题}`，中间 `< 日期 >`，右侧 `搜索 → RAG+CHAT → 灯泡`。
- 页面标题映射：拾微=白描、回顾=回顾、沉淀=沉淀、明悟=明悟。
- 副标题映射：拾微/回顾/沉淀显示“今日 X 字”；明悟不显示字数副标题。
- 设置图标（≡）点击后导航到系统设置页；标题文字不再可点击。
- 移除原顶部栏中的标签图标、随机漫步图标、关于弹窗触发（标题点击）。
- 日期组件居中，保留左右箭头切换日期，点击日期打开日期选择器。
- RAG+CHAT 入口（MessageSquare）点击后进入 Copilot 模式/页面；灯泡入口功能保持不变。
- 底部 TabBar 维持四个主 Tab，但“沉淀 / 明悟”图标按新语义替换。

### Seam 2：系统设置页重构

- 将系统设置页从横向标签切换为**左侧竖排菜单 + 右侧内容区**。
- 左侧菜单项（按顺序）：对话模型、语音朗读、向量与语义、数据管理、提示词配置、标签设置、关于。
- 桌面端：左侧菜单常驻，右侧内容区自适应滚动。
- 移动端：左侧菜单以 push 抽屉动画呈现（参考 blinko 的 `react-burger-menu` push 模式，0.3s 过渡），点击菜单项后抽屉关闭。
- “关于”标签页内容：应用图标、名称、版本号、作者、简介、检查更新并重载、提交反馈（GitHub Issues）。
- 语言切换组件改为横向并排胶囊按钮，统一宽高，当前语言高亮。
- 标签设置菜单项：将原独立标签管理页的内容**内嵌**到系统设置右侧内容区，去掉原独立页的顶部返回栏。

### Seam 3：RAG+CHAT（Copilot）导航重构

- 移除页面内原有的“对话 / 历史会话”顶部标签切换。
- 在页面顶部引入一行横向导航：`RAG`、`CHAT`、`历史`。
- 导航映射：
  - `RAG`：原 RAG 问答模式（基于本地向量检索 + LLM）。
  - `CHAT`：原通用 Chat 模式（纯 LLM 对话）。
  - `历史`：原历史会话列表。
- 切换 `RAG`/`CHAT` 时仍按现有逻辑清空当前会话并新建会话（mode per conversation）。
- 历史列表项继续显示每条会话的 mode 标签（RAG/CHAT）。
- `RAG` 模式下保留原有筛选胶囊行（记录 / 日记 / 回顾 / 洞察、日期、日记模板），内容保持不变。
- 全局顶部栏规则仍适用于该页面：左侧 `[≡] {标题}`，右侧搜索/RAG+CHAT/灯泡。

### Seam 4：多媒体记录卡片渲染

- 拾微记录卡片根据 `RawLog` 中的附件元数据和摘要数据渲染媒体区。
- 图片/视频：
  - 单张：16:9 比例，宽度撑满卡片，高度自适应，`object-fit: cover`。
  - 多张：2×2 网格，每格 16:9，最多显示 4 张；超出时最后一张显示 “+N” 覆盖层。
  - 点击单张或网格中的图片进入灯箱/详情；点击 “+N” 进入该记录的详情页/完整视图。
  - 视频首帧作为缩略图，叠加播放图标；点击播放。
- 音频：
  - 以纵向列表展示每个音频的播放器控件（播放/进度/音量/时长）。
  - 摘要文本使用 `RawLog.content`（STT 结果）。
- 摘要区：
  - 位于媒体区下方，字号与文字拾微一致，次要文本色，最多 3 行，超出截断。
  - 生成中显示占位文案；失败显示“摘要生成失败 · 重新生成”。
  - 重新生成以单个附件为单位：图片/视频按钮在缩略图右下角，音频按钮在播放器控件右侧下方。
- 数据已存在（`AttachmentMeta.summary`、`RawLog.attachment_summary`、`RawLog.content`），本 seam 主要是前端渲染层改造。

### Seam 5：沉淀卡片限高与缩略图

- 沉淀卡片（时间线与瀑布流两种视图）需要支持折叠态与展开态。
- 折叠态最大高度：时间线 7 行，瀑布流 12 行（按正文行高计算）。
- 超出部分使用渐变遮罩 + “展开”按钮提示。
- 单击卡片切换展开/折叠；双击进入编辑。
- 单击/双击判别：约 300ms 内第二次点击视为双击，取消单击动作。
- 卡片内多媒体缩略图统一 1:1 比例：
  - 时间线：一行最多 3 个。
  - 瀑布流：一行最多 2 个。
  - 超出显示 “+N” 或进入详情。

### Seam 6：输入工具栏与附件上传面板

- **沉淀页富文本输入框工具栏**增加以下按钮（从左到右）：
  - 通用上传文件按钮：唤起文件选择器，accept `image/*,audio/*,video/*`，多选，走现有附件处理流程。
  - 超链接按钮：弹出输入框填写 URL 与显示文本，插入 Markdown 链接。
  - 麦克风按钮：点击开始录音，再次点击结束，走 STT 流程，转写文本插入光标处。
  - `#` 标签按钮：点击插入 `#` 或弹出标签选择器。
  - `…` 更多按钮：展开菜单包含表格、代码块、内联代码、导出、预览 Markdown。
- **拾微页底部附件上传面板**：
  - 点击附件按钮后，从底部上滑弹出半屏面板。
  - 面板选项：相册（图片）、音频、视频、链接、文件、取消。
  - 使用 lucide-react 大图标 + 文字标签。
  - 网格布局：第一行 3 个（相册/音频/视频），第二行 2 个功能项 + 取消按钮。
  - 取消按钮样式稍弱（灰色）。
  - 点击遮罩/外部或取消按钮关闭面板。

### Seam 7：提示词配置合并

- 设置 → 提示词配置中调整区块：
  1. **日记回顾生成 Prompt**（合并原日记生成 + 回顾生成）：5 槽标签 `日记 / 回顾 / 自定义 1 / 自定义 2 / 自定义 3`，保留“自动生成选中”复选框。
  2. **明悟和洞察生成 Prompt**（合并原明悟生成 + 洞察生成）：5 槽标签 `明悟 / 洞察 / 自定义 1 / 自定义 2 / 自定义 3`，保留“自动生成选中”复选框。
  3. **日记回顾一句话摘要生成 Prompt**（合并原日记摘要 + 回顾摘要）。
  4. **明悟和洞察一句话摘要生成 Prompt**（由原洞察摘要扩展，补充明悟默认摘要）。
- 数据迁移：
  - 读取 persisted settings 时，将旧的 `mingwuPrompt`/`insightPrompt` 合并到新结构。
  - 将旧的 `diarySummaryPrompt`/`summaryPrompt`（回顾摘要）合并到新结构。
  - 旧的单字段保留只读兼容，保存时写入新结构。
- Zustand store 中引入新的统一字段，并更新 `merge`/`migrate` 逻辑。

### Seam 8：TTS 外部 API 配置

- 系统设置新增“语音朗读”独立菜单项，位于“对话模型”之后。
- 语音朗读服务二选一：浏览器内置（Web Speech）/ 外部 TTS API。
- 选择浏览器内置时隐藏外部 API 字段；选择外部 API 时展开：
  - Provider：Gemini / 火山引擎（暂时）。
  - API Key
  - Base URL
  - Model
  - Voice / Speaker ID
  - 默认朗读语言
  - 朗读语速
- UI 样式参考对话模型当前配置卡片。
- 后端 `/api/tts` 端点接收 `{ text, lang }` 并返回音频 blob；前端根据 Provider 调用对应后端逻辑。
- 保留现有 Web Speech 调用路径作为默认 fallback。

### Seam 9：i18n 文案修正

- 全量扫描 `zh.ts` / `en.ts` 中所有菜单、按钮、Tooltip 文案 key。
- 修正所有 key 直接暴露导致的中英混编问题（如 `view.copyContent编辑内容`）。
- 确保右键/长按菜单、卡片操作栏、导出导入按钮等严格按当前语言环境显示。
- 新增需求中涉及的新文案统一写入 i18n 字典，不在组件中硬编码。

## Testing Decisions

### 测试原则

- 只测试**外部可观察行为**，不测试实现细节（如具体组件内部状态、CSS 类名）。
- 优先复用现有测试 seams；新增测试覆盖用户交互的关键路径。
- 移动端与桌面端分别验证布局、抽屉动画、触摸交互。

### 各 seam 测试重点

1. **全局 Layout/顶部栏**
   - 验证四个主 Tab 的顶部栏结构一致：设置图标、标题、副标题、日期、搜索、RAG+CHAT、灯泡。
   - 验证标题不可点击，设置图标可导航到 `/settings`。
   - 验证 TabBar 图标替换后正确高亮当前 Tab。

2. **系统设置页**
   - 验证左侧菜单切换右侧内容区。
   - 验证移动端抽屉打开/关闭、点击菜单项后关闭。
   - 验证语言切换按钮排版正确且生效。
   - 验证“关于”页显示版本、作者、检查更新、反馈链接。
   - 验证标签设置内嵌后可进行增删改查/重命名/合并/删除。

3. **RAG+CHAT 导航**
   - 验证 `RAG / CHAT / 历史` 导航切换后显示对应内容。
   - 验证 RAG 模式下筛选胶囊、日期筛选、日记模板筛选继续工作。
   - 验证切换 RAG/CHAT 会新建会话，历史列表保留原 mode 标签。

4. **多媒体卡片**
   - 验证图片/视频缩略图正确渲染，单张与多张布局符合规格。
   - 验证音频播放器渲染且摘要显示。
   - 验证摘要失败时显示重试按钮，重试后更新摘要。
   - 验证点击 “+N” 进入详情/灯箱。

5. **沉淀卡片**
   - 验证长文卡片折叠到指定高度并显示“展开”。
   - 验证单击展开/收起，双击进入编辑（不触发展开）。
   - 验证时间线/瀑布流缩略图一行 3/2 个，1:1 比例。

6. **输入工具栏与附件面板**
   - 验证沉淀工具栏各按钮触发对应行为：上传文件、插入链接、录音转写、插入标签、更多菜单。
   - 验证拾微附件面板从底部上滑，六个选项布局正确，点击遮罩/取消关闭。

7. **提示词配置合并**
   - 验证旧设置迁移后，新配置页正确显示合并后的 Prompt。
   - 验证保存后自动生成逻辑读取新字段。
   - 验证切换语言后 Prompt 仍按 per-language 正确显示。

8. **TTS 外部 API**
   - 验证选择外部 API 后展开配置字段，选择浏览器内置后隐藏。
   - 验证保存配置后，TTS 朗读请求正确携带 Provider/Key/Model 等参数。

9. **i18n 文案**
   - 验证中文环境下所有菜单/按钮显示中文，无 key 暴露。
   - 验证英文环境下对应显示英文。

### 测试类型与工具

- **组件/单元测试**：使用项目现有测试框架（Vitest + React Testing Library，若已配置），覆盖纯 UI 交互（折叠展开、导航切换、抽屉开关、语言切换）。
- **端到端测试**（可选但推荐）：使用 Playwright/Cypress 覆盖跨页面流程（顶部栏 → 设置 → 标签设置 → 保存 → 返回 → 拾微上传附件）。
- **视觉回归测试**：因涉及大量 UI 改动，建议对顶部栏、设置页、多媒体卡片、沉淀卡片、附件面板做快照对比。

## Further Notes

- **移动端红线**：所有新引入的滚动区域必须使用局部 `overflow-y-auto` 容器；全局 `html/body` 保持 `overflow: hidden` + `overscroll-behavior: none`。附件上滑面板、设置抽屉、日期选择器均需遵守。
- **图标库**：顶部栏、设置页、附件面板等继续使用 `lucide-react`；底部 TabBar 继续使用 `@phosphor-icons/react`。
- **数据兼容性**：本次改动涉及 Settings store 版本升级（从 v11 到 v12），需在 `migrate` 中处理 Prompt 合并与 TTS 外部配置字段的初始化。
- **性能注意**：多媒体摘要重试以单个附件为单位，避免整批重试；沉淀卡片折叠态不应预渲染完整 Markdown，可使用 `line-clamp` 或最大高度截断。
- **无障碍**：顶部栏按钮保留 `aria-label`；折叠/展开按钮需可被屏幕阅读器识别；附件面板选项需有清晰的焦点状态。
- **后续拆分建议**：本 PRD 可拆为 9 个独立 issue（对应上述 9 个 seams），按以下顺序并行/串行实施：
  1. Seam 9（i18n 文案修正）—— 低风险，可先行。
  2. Seam 1（全局 Layout/顶部栏）+ Seam 2（系统设置重构）—— 相互依赖，建议同批次。
  3. Seam 3（RAG+CHAT 导航）+ Seam 8（TTS 外部 API）—— 相对独立。
  4. Seam 4（多媒体卡片）+ Seam 5（沉淀卡片）—— 都与卡片渲染相关，可同批次。
  5. Seam 6（输入工具栏与附件面板）+ Seam 7（提示词配置合并）—— 输入/设置相关。
