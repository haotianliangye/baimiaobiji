# Seam 1：统一全局 Layout 与顶部标题栏

## Problem Statement

当前四个主 Tab（碎屑、回顾、沉思、明悟）的顶部栏结构不一致，入口混乱：
- 顶部同时存在“白描笔记”标题、次级导航栏、设置图标、标签图标、随机漫步图标、Copilot 入口、灯泡入口，视觉重心分散。
- 点击“白描”标题会触发设置弹窗，容易误触。
- 底部 TabBar 中“沉思 / 明悟”图标与页面语义不匹配。

## Solution

统一四个主 Tab 的顶部标题栏结构，规范入口与图标语义：
- 左侧：`[≡] {页面标题} · {副标题}`
- 中间：`< 日期 >`，保留左右箭头切换，点击打开日期选择器。
- 右侧：`搜索 → RAG+CHAT → 灯泡`
- 标题不再可点击，仅设置图标（≡）打开系统设置页。
- 移除顶部栏中的标签图标、随机漫步图标、关于弹窗触发。
- 底部 TabBar“沉思 / 明悟”图标按新语义替换。

## User Stories

1. 作为用户，我希望顶部标题栏在四个主 Tab 间保持一致，以便在不同板块间快速建立操作预期。
2. 作为用户，我希望只有设置图标（≡）能打开系统设置，而不是点击标题文字，以避免误触。
3. 作为用户，我希望四个主 Tab 的顶部栏右侧始终有搜索、RAG+CHAT 入口和灯泡，以便随时使用全局能力。
4. 作为用户，我希望碎屑/回顾/沉思页顶部显示“今日 X 字”，以便一眼看到当日输入量。
5. 作为用户，我希望日期居中并保留左右箭头切换，同时点击日期仍能打开日期选择器，以便快速跳转日期。
6. 作为用户，我希望“白描”替代“白描笔记”作为标题，并保留“白描”的简洁品牌感。
7. 作为用户，我希望底部 TabBar 中“沉思 / 明悟”图标与页面语义匹配，以便快速识别当前板块。

## Implementation Decisions

- 统一四个主 Tab 的顶部标题栏结构：左侧 `[≡] {页面标题} · {副标题}`，中间 `< 日期 >`，右侧 `搜索 → RAG+CHAT → 灯泡`。
- 页面标题映射：碎屑=白描、回顾=回顾、沉思=沉思、明悟=明悟。
- 副标题映射：碎屑/回顾/沉思显示“今日 X 字”；明悟不显示字数副标题。
- 设置图标（≡）点击后导航到系统设置页；标题文字不再可点击。
- 移除原顶部栏中的标签图标、随机漫步图标、关于弹窗触发（标题点击）。
- 日期组件居中，保留左右箭头切换日期，点击日期打开日期选择器。
- RAG+CHAT 入口（MessageSquare）点击后进入 Copilot 模式/页面；灯泡入口功能保持不变。
- 底部 TabBar 维持四个主 Tab，但“沉思 / 明悟”图标按新语义替换：
  - 沉思使用原明悟图标（更符合沉思含义）。
  - 明悟使用 `sun-dim` 图标。

## Testing Decisions

- 验证四个主 Tab 的顶部栏结构一致：设置图标、标题、副标题、日期、搜索、RAG+CHAT、灯泡。
- 验证标题不可点击，设置图标可导航到 `/settings`。
- 验证 TabBar 图标替换后正确高亮当前 Tab。
- 验证移动端与桌面端布局一致，无元素截断。

## Out of Scope

- 不改动系统设置页内部结构（见 Seam 2）。
- 不改动 RAG+CHAT 页面内部导航（见 Seam 3）。
- 不改动标签管理功能本身，仅移除顶部栏入口（见 Seam 2）。
