# 需求 7：图片/视频全屏预览适配

## Problem Statement

图片点击打开但预览层没做好屏幕自适应，会突破应用容器变真正全屏；视频只能在卡片缩略图内播放，没有放大到屏幕大小的预览。

（PRD Problem Statement 第 7 条：图片/视频预览层不适配，图片突破应用容器变真正全屏、视频仅缩略图播放。）

## Solution

图片/视频全屏预览：应用容器内（`max-w-md`）全屏，支持左右滑动/pinch 缩放/双击放大/多关闭方式。统一图片与视频的点击预览体验，点击缩略图后在应用界面内全屏打开，做好自适应，不容器外溢。

（PRD Solution 第 7 条：图片/视频全屏预览--应用容器内全屏，支持左右滑动/pinch 缩放/双击放大/多关闭方式。）

## User Stories

32. 作为用户，我希望点击图片在应用容器内全屏预览（不突破 max-w-md），以便看清。
33. 作为用户，我希望点击视频在应用容器内全屏播放（非仅缩略图），以便看清。
34. 作为用户，我希望左右滑动切换同一条记录的多个图片/视频，以便浏览。
35. 作为用户，我希望 pinch 缩放和双击放大/缩小，以便看细节。
36. 作为用户，我希望点遮罩/右上角 ×/电脑端 ESC 都能关闭预览，以便灵活关闭。
37. 作为用户，我希望视频预览自动播放、不静音、循环，以便连贯观看。
38. 作为用户，我希望预览打开时隐藏顶部标题栏和底部 TabBar，以便适配多种比例。

## Implementation Decisions

> 来源：PRD Implementation Decisions「需求 7」章节 + 需求基线「需求 7」已确认细节 + 相关跨需求约束。

### 核心行为
- 图片点击后应用容器内（`max-w-md`）全屏预览，不突破容器。
- 视频点击后应用容器内全屏播放（非仅缩略图）。

### 覆盖范围
- 需支持全屏预览的页面：**拾微 + 沉淀**。
- 回顾/明悟无图片/视频，无需改。

### 预览层范围
- 预览层占满整个 `max-w-md` 容器，不超设备屏幕。
- 预览打开时隐藏顶部标题栏和底部 TabBar，适配 9:16/16:9/3:4 等比例。

### 多图/多视频切换
- 支持左右滑动切换同一条记录内多个图片/视频。

### 缩放
- 支持 pinch 缩放。
- 支持双击放大/缩小。

### 关闭
- 点击遮罩 / 点击右上角 × / 电脑端按 ESC，三方式并存。

### 视频播放
- 自动播放、不静音（默认有声）、循环播放。

### 相关跨需求约束
- **移动端红线**：全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`；全屏预览作为新滚动区用局部 `overflow-y-auto`，不依赖 body 滚动。
- **多媒体卡片细节（G9/G10）**：混合媒体布局顺序为图片/视频 2×2 网格 -> 音频纵向列表 -> AI 摘要文本；点击 +N 进详情/完整视图（含全部附件 + 完整摘要 + 原始内容）。本需求负责点击缩略图后的全屏预览/播放体验，与卡片网格布局的点击入口衔接。
- **附件存储**：仅读取已有 attachments 字段，不改附件存储 schema。

## Testing Decisions

- seam: `multimedia.test.ts`（需求 7：图片/视频全屏预览--滑动/缩放/关闭）
- 原则：只测外部可观察行为（UI 渲染/交互/可见结果），不测实现细节（CSS 类名/内部状态）
- 测试重点（PRD Testing 第 7 条）：全屏预览--容器内全屏、视频播放、左右滑动、pinch/双击缩放、三方式关闭、隐藏 header/TabBar。即：
  - 图片点击在应用容器内（`max-w-md`）全屏预览，不突破容器。
  - 视频点击在应用容器内全屏播放（非仅缩略图），自动播放、不静音、循环。
  - 多图/多视频左右滑动切换。
  - pinch 缩放 + 双击放大/缩小。
  - 三方式关闭：点遮罩 / 右上角 × / 电脑端 ESC。
  - 预览打开时隐藏顶部标题栏和底部 TabBar。
  - 覆盖拾微 + 沉淀页（回顾/明悟无图片视频无需改）。

## Out of Scope

- 不改数据同步/云备份/OAuth
- 不涉端侧 AI 迁移
- 不改 AI 生成算法
- 不改附件存储 schema（仅读取已有 attachments 字段）
- 不覆盖回顾/明悟页（无图片/视频，无需全屏预览）
- 不改多媒体卡片网格布局本身（2×2 网格/+N 详情由跨需求约束 G9/G10 规定，卡片布局渲染不在本需求改）
- 不改图片/视频附件的上传/新增/删除流程（附件增删属需求 3 编辑弹窗职责）
