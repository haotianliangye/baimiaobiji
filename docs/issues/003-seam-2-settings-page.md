# Seam 2：系统设置页重构

## Problem Statement

当前系统设置页采用横向标签，模块一多即拥挤；语言切换按钮排版不统一；“关于”信息以弹窗形式存在，与设置页割裂；标签管理有独立页面，导致顶部栏入口冗余。

## Solution

将系统设置页重构为左侧竖排菜单 + 右侧内容区：
- 桌面端左侧菜单常驻，右侧内容区自适应滚动。
- 移动端左侧菜单以 push 抽屉动画展开，点击菜单项后抽屉关闭。
- 新增“语音朗读”和“标签设置”菜单项。
- 新增“关于”标签页，并入原关于弹窗内容。
- 语言切换改为横向并排胶囊按钮，统一宽高，当前语言高亮。
- 将原独立标签管理页的内容内嵌到系统设置右侧内容区。

## User Stories

1. 作为用户，我希望系统设置采用左侧菜单 + 右侧内容区的布局，以便在模块增多时仍能清晰导航。
2. 作为用户，我希望系统设置在桌面端左侧菜单常驻、在移动端以 push 抽屉动画展开，以便在不同设备上都有良好体验。
3. 作为用户，我希望“关于”信息（版本、作者、检查更新、反馈）放在系统设置内的“关于”标签页，而不是单独弹窗，以便统一查找。
4. 作为用户，我希望语言切换按钮“中文 / English”排版统一为一行且高亮当前语言，以便界面更整齐。
5. 作为用户，我希望标签管理从顶部栏图标迁移到系统设置内的“标签设置”，以便顶部栏更简洁。
6. 作为用户，我希望系统设置中新增“语音朗读”菜单项，以便配置 TTS 服务（见 Seam 8）。

## Implementation Decisions

- 将系统设置页从横向标签切换为**左侧竖排菜单 + 右侧内容区**。
- 左侧菜单项（按顺序）：
  1. 对话模型
  2. 语音朗读
  3. 向量与语义
  4. 数据管理
  5. 提示词配置
  6. 标签设置
  7. 关于
- 桌面端：左侧菜单常驻，右侧内容区自适应滚动。
- 移动端：左侧菜单以 push 抽屉动画呈现（参考 blinko 的 `react-burger-menu` push 模式，0.3s 过渡），点击菜单项后抽屉关闭。
- “关于”标签页内容：应用图标、名称、版本号、作者、简介、检查更新并重载、提交反馈（GitHub Issues）。
- 语言切换组件改为横向并排胶囊按钮，统一宽高，当前语言高亮。
- 标签设置菜单项：将原独立标签管理页的内容**内嵌**到系统设置右侧内容区，去掉原独立页的顶部返回栏。

## Testing Decisions

- 验证左侧菜单切换右侧内容区。
- 验证移动端抽屉打开/关闭、点击菜单项后关闭。
- 验证语言切换按钮排版正确且生效。
- 验证“关于”页显示版本、作者、检查更新、反馈链接。
- 验证标签设置内嵌后可进行增删改查/重命名/合并/删除。
- 验证所有新增滚动区域使用局部 `overflow-y-auto`，全局 `html/body` 保持 `overflow: hidden`。

## Out of Scope

- 提示词配置内容合并（见 Seam 7）。
- TTS 外部 API 配置逻辑（见 Seam 8）。
- 不改动对话模型、向量语义、数据管理现有功能，仅调整布局。
