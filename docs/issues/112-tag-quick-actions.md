# 标签快捷操作菜单：置顶、编辑、移除、删除

## Parent

- #110 UI/UX 重构 v3 跟进（需求补漏）

## What to build

在设置抽屉的「所有标签」列表中，为每个标签项增加快捷操作能力：

1. 每个标签右侧显示两个图标按钮：展开/收起子标签的箭头（`ChevronDown` / `ChevronUp`），以及三个点「更多」菜单（`MoreVertical`）。
2. 「更多」菜单为弹出式动作菜单，包含：置顶、编辑名称和图标、仅移除标签、删除标签和笔记。
3. 危险操作（仅移除标签、删除标签和笔记）使用红色/玫瑰色警示。
4. 扩展 `TagDef` schema 与 `tags.store.ts`，支持 `pinned` / `sort_order` / `icon` 字段的持久化：置顶标签排在列表最前，编辑名称和图标后更新到数据库。
5. 「仅移除标签」从所有关联记录中移除该标签但不删除记录，并需确认弹窗；「删除标签和笔记」级联删除标签定义及所有带该标签的记录，并需二次确认。

该 slice 需要贯穿 schema、store、UI、持久化与测试，是端到端的垂直切片。

## Acceptance criteria

- [ ] 每个标签项右侧有展开/收起箭头与三个点「更多」菜单图标
- [ ] 点击「更多」弹出菜单，含「置顶」「编辑名称和图标」「仅移除标签」「删除标签和笔记」四项
- [ ] 危险操作项在菜单中以警示色区分
- [ ] 点击菜单外部或滚动时菜单关闭
- [ ] 置顶后标签排在列表最前，刷新后仍保持置顶状态
- [ ] 编辑名称和图标后，数据库与 UI 同步更新
- [ ] 「仅移除标签」保留记录，仅移除标签关联，并弹出确认
- [ ] 「删除标签和笔记」删除标签定义并级联删除关联记录，需二次确认
- [ ] `TagDef` schema 与 `tags.store.ts` 支持 `pinned` / `sort_order` / `icon`
- [ ] `tests/foundation-migration.test.ts` 覆盖标签菜单的主要外部行为
