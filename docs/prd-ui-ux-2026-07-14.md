## Problem Statement

白描笔记当前 UI/UX 存在多处体验断层：

1. 顶部全局 header 的"今日 X 字"副标题与各页面内部 header 重复统计，信息分散且占用顶部空间。
2. 随机漫步作为固定全屏覆盖层，卡片尺寸写死，桌面宽屏留白过多、手机端空间利用不足。
3. 拾微编辑仍用底部全屏面板 + 纯文本 textarea，不支持多媒体附件增删，与创建时的多媒体能力不匹配。
4. 编辑拾微/回顾需长按/右键菜单或展开后点编辑按钮，操作路径长。
5. 明悟 Tab 用发光/灯泡类图标，与"明悟"语义不直观。
6. 四个主 Tab 各有独立次级导航栏，顶部堆叠；沉淀/明悟中间区域缺下拉胶囊切换器。
7. 图片/视频预览层不适配，图片突破应用容器变真正全屏、视频仅缩略图播放。
8. 拾微附件上传直接调系统文件选择器，缺中间面板选上传类型。
9. 系统设置左侧固定菜单 + 右侧分栏不够紧凑，移动端体验欠佳。

## Solution

1. **统计小字下移**：把"今日 X 字"从顶部 header 移到各输入/操作模块左上方，统一"今日 X 条 X 字"，清理重复展示。
2. **随机漫步主内容区模式**：限制在应用容器内，swiper 扇形堆叠滑动，保留顶部 header 和底部 TabBar，编辑按钮弹 RichEditor。
3. **拾微编辑弹窗多媒体化**：居中弹窗 + RichEditor，支持多媒体附件增删，复用创建时的附件处理流程。
4. **拾微/回顾双击编辑**：双击卡片直接进编辑（拾微弹窗、回顾 inline）。
5. **明悟 Tab 图标改 Sun**：TabBar + 明悟页 header + 全局旧图标处统一换 Sun。
6. **顶部标题栏重构**：统一四 Tab 顶部栏（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡），沉淀下拉胶囊切换器、明悟时间范围下拉胶囊。
7. **图片/视频全屏预览**：应用容器内全屏，支持左右滑动/pinch 缩放/双击放大/多关闭方式。
8. **拾微附件上传动画面板**：点击附件按钮从底部上滑网格面板，6 选项。
9. **设置页重构**：抽屉 + 全页详情 + 顶部横向导航栏（推翻 Seam 2 的左侧分栏）。

## User Stories

### 统计小字下移与统一
1. 作为用户，我希望顶部全局 header 不显示"今日 X 字"副标题，以便顶部简洁。
2. 作为用户，我希望拾微页底部输入框左上方显示"今日 X 条 X 字"，以便一眼看到当日输入量。
3. 作为用户，我希望回顾页卡片列表底部显示"今日 X 条 X 字"，以便看到当日回顾量。
4. 作为用户，我希望沉淀页输入框左上方显示"今日 X 条 X 字"，以便看到当日沉淀量。
5. 作为用户，我希望明悟页不显示任何"今日"统计，以便聚焦洞察内容。
6. 作为用户，我希望 0 条、非今日、按钮禁用时统计仍显示，以便布局稳定。

### 随机漫步屏幕自适应
7. 作为用户，我希望随机漫步在应用容器内显示（非全屏覆盖），以便保留顶部 header 和底部 TabBar。
8. 作为用户，我希望随机漫步卡片用扇形堆叠滑动效果，以便流畅浏览。
9. 作为用户，我希望点击灯泡/×/底部 Tab 都能退出随机漫步，以便灵活退出。
10. 作为用户，我希望随机漫步编辑按钮弹出 RichEditor 编辑弹窗（不跳转页面），以便快速编辑任意类型记录。
11. 作为用户，我希望随机漫步在手机和桌面都有合理的卡片占用比例，以便不同屏幕都舒适。
12. 作为用户，我希望底部操作栏是单排按钮（已阅/标签/编辑/复制/删除/换一批），"下一张"靠左右滑动，以便操作集中。

### 拾微编辑弹窗多媒体化
13. 作为用户，我希望编辑拾微时弹出居中弹窗（非底部全屏面板），以便专注编辑。
14. 作为用户，我希望编辑弹窗用 RichEditor（含 Markdown 工具栏/上传/超链接/录音/标签/预览），以便富文本编辑。
15. 作为用户，我希望编辑弹窗能新增图片/音频/视频/链接/文件附件，以便拾微创建后仍可补充多媒体。
16. 作为用户，我希望编辑弹窗能删除已有附件，以便管理拾微附件。
17. 作为用户，我希望编辑弹窗底部有删除（整条）/取消/保存按钮，以便操作。

### 拾微/回顾卡片双击编辑
18. 作为用户，我希望双击拾微卡片直接打开编辑弹窗，以便快速编辑。
19. 作为用户，我希望双击回顾卡片触发 inline 编辑（textarea 获得焦点），以便快速编辑。
20. 作为用户，我希望多选模式下双击不触发编辑，以免误操作。
21. 作为用户，我希望拾微附件区单击仍打开灯箱/详情，双击只在非附件区触发编辑，以便两个行为不冲突。

### 明悟 Tab 图标改为 Sun
22. 作为用户，我希望底部 TabBar 明悟 tab 用 Sun 图标，以便图标与"明悟"语义匹配。
23. 作为用户，我希望明悟页内部 header 图标也用 Sun，以便全局一致。
24. 作为用户，我希望所有用到旧明悟图标处（空状态/加载占位/生成按钮等）统一换 Sun，以便无遗漏。

### 顶部标题栏与次级导航重构
25. 作为用户，我希望四个主 Tab 顶部栏统一结构（左 ≡ 标题 / 中 日期或胶囊 / 右 搜索·RAG·灯泡），以便一致体验。
26. 作为用户，我希望标题文字不可点击，仅 ≡ 图标打开系统设置，以免误触。
27. 作为用户，我希望拾微/回顾中间是日期选择器（月-日 + 左右箭头 + 点击弹选择器），以便切日期。
28. 作为用户，我希望沉淀中间是瀑布流/时间线下拉胶囊切换器，以便切视图。
29. 作为用户，我希望明悟中间是时间范围下拉胶囊（今日/本周/本月/本季度/半年/一年/自选），以便选范围。
30. 作为用户，我希望日期字号比标题略小，以便视觉层次。
31. 作为用户，我希望沉淀胶囊文案是"瀑布流/时间线"仅文字无图标，以便简洁。

### 图片/视频全屏预览适配
32. 作为用户，我希望点击图片在应用容器内全屏预览（不突破 max-w-md），以便看清。
33. 作为用户，我希望点击视频在应用容器内全屏播放（非仅缩略图），以便看清。
34. 作为用户，我希望左右滑动切换同一条记录的多个图片/视频，以便浏览。
35. 作为用户，我希望 pinch 缩放和双击放大/缩小，以便看细节。
36. 作为用户，我希望点遮罩/右上角 ×/电脑端 ESC 都能关闭预览，以便灵活关闭。
37. 作为用户，我希望视频预览自动播放、不静音、循环，以便连贯观看。
38. 作为用户，我希望预览打开时隐藏顶部标题栏和底部 TabBar，以便适配多种比例。

### 拾微附件上传入口动画面板
39. 作为用户，我希望点附件按钮从底部上滑面板选类型，以便直观选择。
40. 作为用户，我希望面板有相册/音频/视频/链接/文件/取消 6 项大图标，以便覆盖所有类型。
41. 作为用户，我希望点取消/面板外部/下滑都能关闭面板，以便灵活退出。
42. 作为用户，我希望面板上滑时把输入栏和主内容向上推，以便层级清晰。

### 设置页展示方式重构
43. 作为用户，我希望点 ≡ 从左侧滑出抽屉（含设置菜单项 + 所有标签区块），以便统一查看。
44. 作为用户，我希望点抽屉菜单项进全屏设置详情页（顶部横向导航栏），以便专注配置。
45. 作为用户，我希望横向导航栏切换设置项（动画过渡 + 左右滑动），以便快速切换。
46. 作为用户，我希望横向导航栏用胶囊高亮当前项，以便明确位置。
47. 作为用户，我希望桌面端也用抽屉 + 全页模式（不保留左右分栏），以便跨设备一致。
48. 作为用户，我希望标签设置作为横向导航 tab + 抽屉"所有标签"快捷入口，以便便捷访问。
49. 作为用户，我希望标签多时仅"所有标签"区块内部滚动，抽屉其他部分不动，以便操作稳定。
50. 作为用户，我希望全屏详情页直接关闭设置（不返回抽屉），以便快速退出。

## Implementation Decisions

### 需求 1：统计小字下移与统一
- 新增共享统计组件，数据计算保留各页面内部，通过 props 传入。
- 顶部全局 header 移除"今日 X 字"副标题。
- 拾微页：移除页面内 header 统计 pill，底部输入框左上方新增"今日 {count} 条 {chars} 字"，按当前查看日期统计 raw_logs（count=条目数，chars=countChars(content) 含转写文本不含图片视频摘要）。
- 回顾页：移除页面内 header 统计 pill，卡片列表底部新增统计（空状态放"AI 智能整理"按钮上方，有内容放"AI 智能整理(追加)"按钮上方），口径 countChars(ai_editorial || ai_review)。
- 沉淀页：移除页面内 header"总条数·总字数"，底部输入左上方新增统计，按真实今日 created_at 统计 thoughts。
- 明悟页：不显示任何统计。
- 样式：text-[11px] / text-stone-400 / font-medium / 模块左上方 mb-1.5。
- i18n 新增 common.todayStats（zh 今日 {count} 条 {chars} 字 / en Today {count} notes · {chars} chars）。

### 需求 2：随机漫步屏幕自适应
- 随机漫步从 fixed 全屏覆盖层改为 main 内容区渲染（类似 Copilot 模式），限制在 max-w-md 容器内，保留顶部 header 和底部 TabBar。
- 新增依赖 swiper ^11.2.6，卡片区域用 Swiper + EffectCards 扇形堆叠滑动，包裹现有 WalkItem 卡片内容。
- 全局状态控制显隐，入口仍为 header 灯泡按钮；激活时 header 标题显示随机漫步标题、右侧 × 关闭；TabBar 切换 Tab 退出。
- 底部操作栏单排：已阅/标签/编辑/复制/删除/换一批，"下一张"靠左右滑动。
- 编辑按钮弹 RichEditor 编辑弹窗（不跳转），所有类型统一；raw_logs/thoughts 支持附件编辑，daily_reviews/mingwu 仅文本；保存按类型写回；不显示/不改展示时间。
- 卡片宽度占满容器（父容器左右 px-5），高度 flex-1 填充 header 与操作栏之间。
- 标签保留底部按钮 + 现有 tag sheet，不在编辑弹窗处理。
- 空状态保持居中卡片式，限制在容器内；卡片文字字号不响应式调整。
- 参考 Blinko review 页 swiper 实现。
- **G3 澄清**：随机漫步保留 header 灯泡入口（不迁设置）；顶部栏移除的是标签图标（TagsIcon）非随机漫步；标签管理在设置抽屉（需求 9）。

### 需求 3：拾微编辑弹窗多媒体化
- 移除底部滑上全屏编辑面板，改居中弹窗（遮罩 + backdrop-blur + 居中 + max-w-md + max-h-[88vh] + flex-col，内容区滚动，底部操作栏固定）。
- 用 RichEditor 组件，完整保留 Markdown 工具栏/上传/超链接/录音/标签/预览。编辑时回显原 attachments 数组。audioBlob 保留但不可编辑，删整条记录时一并删。
- 新增附件走 saveAttachmentBlob + 异步摘要（图片/视频后台多模态摘要、音频 STT 拼入 content、链接/文件仅元数据）。删除附件从 attachments 移除并删 store Blob。清空所有图片/视频时 attachment_summary 一并清空。
- 保存条件：content.trim() 非空 或 attachments 非空；保存时 processTags 重新解析 #标签 落库。
- 底部操作栏：左删除（确认删整条）、右取消/保存；字数统计保留左侧。
- 入口：长按/右键"编辑记录" + 需求 4 双击。
- RichEditor 需支持渲染已存附件缩略图（已存 ref 是 store id，加载 Blob 转 object URL）；新增附件内部读 File 为 data URL；保存区分新/旧附件，仅新附件调 saveAttachmentBlob；删旧附件调 attachments.delete(ref)。
- i18n 复用现有 key，无新增。
- **G5 补充**：RichEditor 麦克风按钮录音中显示"录音中"状态。

### 需求 4：拾微/回顾卡片双击编辑
- 拾微页 LogCard 绑定 onDoubleClick（卡片任意区域除操作按钮），双击打开需求 3 的 RichEditor 弹窗；多选模式不响应；附件区单击保留灯箱/详情，双击只在非附件区触发。
- 回顾页内容卡片绑 onDoubleClick，双击触发现有 inline 编辑（展开后 textarea 获得焦点）；多选模式不响应。
- onDoubleClick 事件实现，不引入双击库；处理单击/双击互斥（回顾单击折叠/展开，双击进编辑）；多选父组件统一控制，进入多选移除双击监听。

### 需求 5：明悟 Tab 图标改为 Sun
- TabBar 明悟 tab 图标改 @phosphor-icons/react 的 Sun，粗细 regular，样式（大小/描边/颜色/active-inactive）与其他 tab 一致，不改布局/间距/动画。
- 明悟页内部 header 图标 + 全局所有旧明悟图标处（空状态/加载占位/生成按钮等）一并换 Sun。
- i18n 复用 tabs.mingwu，无新增。

### 需求 6：顶部标题栏与次级导航重构
- 移除四 Tab 各页面次级导航栏，内容合并到统一顶部标题栏。
- 左侧：[≡] {页面标题}。标题随 Tab：拾微=白描、回顾=回顾、沉淀=沉淀、明悟=明悟。仅 ≡ 打开系统设置，标题不可点击。
- 中间：拾微/回顾居中日期选择器 < 日期 >（月-日 + 左右箭头切换 + 点击弹选择器 + 日期字号比标题略小）；沉淀为瀑布流/时间线下拉胶囊切换器（文案"瀑布流/时间线"仅文字）；明悟为时间范围下拉胶囊（复用洞察下拉选项：今日/本周/本月/本季度/半年/一年/自选，弹窗横排网格每行 4 个共两行）。
- 右侧：搜索、RAG、随机漫步灯泡三图标不变。
- 标题栏高度维持不变。
- 仅拾微/回顾显示日期选择器；沉淀/明悟由胶囊替代。

### 需求 7：图片/视频全屏预览适配
- 图片点击后应用容器内（max-w-md）全屏预览，不突破容器；视频点击后应用容器内全屏播放（非仅缩略图）。
- 覆盖：拾微 + 沉淀（回顾/明悟无图片视频无需改）。
- 预览层占满 max-w-md 容器，打开时隐藏顶部标题栏和底部 TabBar，适配 9:16/16:9/3:4。
- 多图/多视频左右滑动切换。
- pinch 缩放 + 双击放大/缩小。
- 关闭：点遮罩 / 右上角 × / 电脑端 ESC。
- 视频：自动播放、不静音、循环。

### 需求 8：拾微附件上传入口动画面板
- 点拾微输入框旁附件按钮，从底部上滑面板（参考微信聊天输入底部面板），网格排列功能图标。
- 面板内容：相册/音频/视频/链接/文件/取消（6 项）。图标用 lucide-react：相册 Image / 音频 Music / 视频 Video / 链接 Link / 文件 FileUp（G8）。
- 面板上滑把输入栏和主内容向上推。关闭：点取消 / 点外部 / 下滑手势。
- 仅拾微页使用。动画时长和缓动按系统默认。

### 需求 9：设置页展示方式重构（推翻 Seam 2 左侧分栏）
- 点顶部 ≡ 从左侧滑出抽屉。抽屉上半部分为原设置菜单项（对话模型/语音朗读/向量与语义/数据管理/提示词配置/标签设置/关于），下半部分新增"所有标签"区块（展示现有标签 + 右上角快捷入口跳标签设置）。
- 点抽屉菜单项跳转全屏设置详情页（不再右侧展开）。详情页顶部横向滚动导航栏（含所有设置项），点击切换详情。
- 标签设置作为横向导航 tab + 抽屉快捷入口双入口。
- 桌面端宽屏也用抽屉 + 全页模式，不保留左右分栏。
- 抽屉设置项图标/排序/分组与现有一致。
- 全屏详情页直接关闭设置（不返回抽屉）。
- 横向导航栏：点击 tab 动画过渡 + 左右滑动切换 + 胶囊/药丸高亮当前。
- 标签多时仅"所有标签"区块内部滚动，抽屉其他部分不跟随；区块高度自适应。

### 跨需求约束

**多媒体卡片细节（G9/G10）**：混合媒体布局顺序为图片/视频 2×2 网格（混排，单格 16:9，最多 4 格，超出 +N）-> 音频纵向列表 -> AI 摘要文本。点击 +N 进详情/完整视图，含全部附件 + 完整摘要 + 原始内容。

**沉淀卡片细节（G7）**：展开后完全展开不限制高度，再次单击收回。

**提示词合并细节（G6）**：字段名 mingwuInsightPrompt（明悟/洞察/custom1-3）/ diaryReviewSummaryPrompt（日记/回顾摘要）/ mingwuInsightSummaryPrompt（明悟洞察摘要）；补默认明悟摘要 prompt（可用洞察摘要副本作默认）。

**数据兼容性**：Settings store 版本 v11 -> v12，在同一 migrate 函数中协调 Prompt 合并 + TTS 外部配置字段初始化。

## Testing Decisions

- 只测外部可观察行为（UI 渲染结构、交互触发、可见结果），不测实现细节（CSS 类名、内部状态、私有方法）。
- 优先复用现有 E2E seam，不新建 seam。用最高 seam（E2E）。
- Prior art：现有 12 个 E2E 测试（Puppeteer + DOM 断言 + 交互触发）。

## Out of Scope

- 不改动数据同步、云备份、OAuth 相关逻辑。
- 不涉及端侧 AI（Capacitor/LiteRT-LM）迁移（见 `docs/on-device-ai-migration.md`）。
- 不改动底层 AI 生成算法、Prompt 内容本身（仅调整配置结构与合并）。
- 不新增 AI 模型服务商（TTS 仅 Gemini/火山引擎）。
- 不改动日记/回顾/明悟/洞察的生成调度与任务队列逻辑（US36 复选框接 generateMingwu 读取选中状态除外）。
- 不改动附件存储 schema（仅读取已有字段）。

## Further Notes

- **移动端红线**：全局 html/body 保持 overflow:hidden + overscroll-behavior:none；所有新滚动区（日期选择器/附件面板/设置抽屉/全屏预览/标签区块）用局部 overflow-y-auto，不依赖 body 滚动。
- **图标库**：顶部栏/设置/面板用 lucide-react；底部 TabBar 用 @phosphor-icons/react。
- **数据兼容性**：Settings store v11 -> v12，migrate 中协调 Prompt 合并 + TTS 字段初始化。
- **性能**：沉淀卡片折叠态不预渲染完整 Markdown（line-clamp 或 max-h 截断）；多媒体摘要重试以单附件为单位。
- **无障碍**：顶部栏按钮保留 aria-label；沉淀折叠/展开按钮可被屏幕阅读器识别；附件面板选项有清晰焦点状态。
- **需求 9 推翻 Seam 2**：之前 Seam 2 实施的"左侧固定菜单 + 右侧内容分栏"被需求 9 推翻，改抽屉 + 全页 + 横向导航。设置页需重做。
- **G3 冲突处理**：随机漫步保留 header 灯泡入口（不迁设置）；顶部栏移除标签图标（TagsIcon），标签管理在设置抽屉。
