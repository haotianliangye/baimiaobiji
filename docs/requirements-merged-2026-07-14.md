## 需求清单

### 需求 1：统计小字下移与统一

**背景**：当前顶部全局 header 显示"白描 · 今日 165 字"作为副标题。各页面内部 header 也重复显示类似统计。信息分散且占用顶部空间。

**目标**：把"今日 X 字"从顶部 header 移到各输入/操作模块的左上方，统一格式为"今日 X 条 X 字"，并清理重复展示。

**具体改动**：

1. **顶部全局 header** (`Layout.tsx`)：移除标题旁的"今日 X 字"副标题。
2. **碎屑页** (`Record.tsx`)：
   - 移除页面内部 header 里的"今日 X 字"统计 pill。
   - 在底部输入框左上方新增"今日 {count} 条 {chars} 字"。
   - 统计当前查看日期 (`?date=`) 的 `raw_logs`：`count` 日志条目数；`chars` = `countChars(log.content)`，包含语音/音频转写文本，不包含图片/视频 AI 摘要。
3. **回顾页** (`Review.tsx`)：
   - 移除页面内部 header 里的"今日 X 字"统计 pill。
   - 在当前查看日期 (`review_date`) 的日记+回顾卡片列表底部新增"今日 {count} 条 {chars} 字"：空状态时放在"AI 智能整理"按钮上方；有内容时放在底部"AI 智能整理(追加)"按钮上方。
   - 统计口径：`countChars(review.ai_editorial || review.ai_review)`。
4. **沉思页** (`Thoughts.tsx`)：
   - 移除页面内部 header 里的"总条数 · 总字数"统计。
   - 在底部快速输入/富文本编辑器左上方新增"今日 {count} 条 {chars} 字"。
   - 按**真实今日**(`created_at`) 统计 `thoughts`。
5. **明悟页** (`Mingwu`)：不显示任何"今日 X 条 X 字"统计。

**样式**：字号 `text-[11px]`、颜色 `text-stone-400`、字重 `font-medium`、位置模块左上方 `mb-1.5`。0 条、非今日、按钮禁用时均显示。

**i18n**：新增 `common.todayStats`（zh: `今日 {count} 条 {chars} 字` / en: `Today {count} notes · {chars} chars`）。

**实现建议**：新增共享组件 `src/components/TodayStats.tsx`，数据计算保留各页面内部，props 传入。

### 需求 2：随机漫步屏幕自适应修复

**背景**：随机漫步(`RandomWalk`)是固定全屏覆盖层(`fixed inset-0`)，卡片区域写死 `w-full max-w-md h-full max-h-[60vh]` 居中。桌面宽屏卡片过小、留白过多；手机端 `max-h-[60vh]` 空间利用不足。

**目标**：修复随机漫步在手机端和桌面端的屏幕适配，让卡片与操作区域在不同屏幕尺寸下都有合理占用比例。

**参考实现**：Blinko 每日回顾页（`D:\DProjects\blinko\app\src\pages\review.tsx`）使用 `swiper` + `EffectCards` 扇形卡片堆叠滑动，`swiper ^11.2.6`，尺寸：手机端 `w-[300px] h-[calc(100vh_-_300px)]`、桌面端 `md:w-[550px]`。

**已确认**：

- 随机漫步改为**主内容区模式**，限制在应用容器(`max-w-md`)内，不再作为全屏覆盖层。
- 手机端和桌面端都保留：顶部 app header（显示"随机漫步"标题，右侧 × 关闭按钮）、底部 TabBar（保留可用，点击 Tab 即退出随机漫步）。
- 退出方式：点击顶部 header 灯泡按钮（再次点击关闭）/ 点击 header 右侧 × / 点击底部任意 Tab。
- 中间卡片区域采用扇形堆叠滑动效果，使用 `swiper` + `EffectCards`，`swiper ^11.2.6`。
- 底部操作栏简化为**单排按钮**：已阅 / 标签 / 编辑 / 复制 / 删除 / 换一批。"下一张"通过左右滑动实现。
- 编辑按钮点击后，**所有类型统一弹出 `RichEditor` 编辑弹窗**，不跳转页面。保存时根据记录类型写回对应表和字段。`raw_logs`/`thoughts` 支持附件编辑；`daily_reviews`/`mingwu` 仅文本。删除按钮。不显示/不修改展示时间。
- 卡片尺寸：宽度占满容器宽度（父容器左右 `px-5` padding）；高度父容器 `flex flex-col`，卡片区域 `flex-1` 填充 header 与底部操作栏之间。
- 标签：保留底部"标签"按钮和现有 tag sheet，不在编辑弹窗内处理。
- 空状态：保持现有居中卡片式，仅限制在应用容器内。卡片内文字大小保持现有字号，不做响应式调整。

**gaps G3 澄清**：随机漫步**保留 header 灯泡入口**（不迁设置）；顶部栏移除的是**标签图标**（TagsIcon）非随机漫步；标签管理在设置抽屉（见需求 9）。

**实现要点**：新增依赖 `swiper ^11.2.6`；`Layout.tsx` 把 `RandomWalk` 从 `fixed` 覆盖层改为 `<main>` 内容区渲染（类似 Copilot 模式）；新增/复用全局状态控制显隐，入口仍为 header 灯泡；激活时 header 标题显示 `randomWalk.title`、右侧 × 关闭；TabBar 切换 Tab 退出；卡片区域 `Swiper` + `EffectCards` 包裹 `WalkItem`；编辑按钮弹 `RichEditor`。

### 需求 3：碎屑编辑弹窗多媒体化

**背景**：碎屑页 (`Record.tsx`) 编辑当前用底部滑上全屏面板 + 纯 `textarea`，不支持多媒体附件增删。创建时已支持图片/音频/视频/链接/文件，但编辑流程仍停留在纯文本。

**目标**：把碎屑编辑从底部全屏面板改为居中弹窗，复用 `RichEditor` + 附件编辑能力，让碎屑在创建后仍可追加、删除多媒体附件。

**具体改动**：

1. **形态变更**：移除底部滑上全屏编辑面板，改为居中弹窗（`fixed inset-0 bg-black/40 backdrop-blur-sm flex items-end sm:items-center justify-center p-3` + `bg-white rounded-2xl w-full max-w-md max-h-[88vh] flex flex-col`），内容区可滚动，底部操作栏固定。
2. **编辑器**：使用 `RichEditor`，完整保留 Markdown 工具栏、上传、超链接、录音、标签提示、预览切换。编辑时回显原 `raw_log.attachments`。原有 `audioBlob` 保留但不可在弹窗内编辑，删除整条记录时一并删除。
3. **附件处理**：新增附件走 `saveAttachmentBlob` + 异步摘要（图片/视频后台多模态摘要、音频走 STT 拼入 content、链接/文件仅元数据）；删除附件从 `attachments` 数组移除并删 `attachments` store 的 Blob；清空所有图片/视频附件时 `attachment_summary` 一并清空。
4. **保存条件**：`content.trim()` 非空 或 `attachments` 非空；保存时 `processTags(editContent)` 重新解析 `#标签` 落库。
5. **底部操作栏**：左侧删除按钮（确认后删整条记录并关闭）、右侧取消/保存；字数统计保留操作栏左侧（`record.totalChars`）。
6. **入口**：保留长按/右键菜单"编辑记录"；与需求 4 联动：双击碎屑卡片也可打开此弹窗。

**i18n**：复用现有 key（`record.editTitle`/`save`/`saving`/`cancel`/`totalChars`/`saveFailed`），无需新增。

**gaps G5 补充**：`RichEditor` 麦克风按钮录音中显示"录音中"状态（点击开始录音，再次点击结束，录音中按钮反馈）。

**实现要点**：`RichEditor` 能渲染已存附件缩略图（已存附件 `ref` 是 store id，需加载 Blob 转 object URL）；新增附件 `RichEditor` 内部读 File 为 data URL；保存时区分新附件（data URL/File）和旧附件（store id），仅新附件调 `saveAttachmentBlob`；删除旧附件调 `db.attachments.delete(ref)`。

### 需求 4：碎屑/回顾卡片双击编辑

**背景**：编辑碎屑和回顾都需长按/右键菜单或展开后点编辑按钮，操作路径长。用户希望像桌面端习惯，双击卡片直接进入编辑。

**目标**：给碎屑页和回顾页内容卡片增加双击编辑能力。

**已确认**：
- **碎屑页**：双击卡片任意区域打开需求 3 的新编辑弹窗，弹窗内所有字段可编辑。
- **回顾页**：维持现有 inline 编辑（卡片展开后内部变 textarea），双击触发该 inline 编辑状态。
- **单击**：回顾卡片保持展开/折叠；碎屑卡片保持现有单击行为。
- **多选模式**：启用多选时禁用双击编辑。

**具体改动**：
1. **碎屑页** (`Record.tsx`)：`LogCard` 绑定 `onDoubleClick`（卡片任意区域，除操作按钮外），双击打开需求 3 的 `RichEditor` 弹窗；多选模式不响应；附件区单击保留打开灯箱/详情，只在非附件元素触发双击编辑。
2. **回顾页** (`Review.tsx`)：回顾内容卡片绑定 `onDoubleClick`，双击触发现有 inline 编辑（展开后 textarea 获得焦点）；多选模式不响应。

**实现要点**：`onDoubleClick` 事件实现，不引入双击库；处理单击与双击互斥（回顾页单击折叠/展开，双击进编辑）；多选状态父组件统一控制，进入多选后移除双击监听。

### 需求 7：图片/视频全屏预览适配修复

**背景**：图片点击打开但预览层没做好屏幕自适应，会突破应用容器变真正全屏；视频只能在卡片缩略图内播放，没有放大到屏幕大小的预览。

**目标**：统一图片与视频的点击预览体验：点击缩略图后在应用界面内（`max-w-md` 容器）全屏打开，做好自适应，不容器外溢。

**已确认**：

1. **核心行为**：图片点击后应用容器内全屏预览，不超 `max-w-md`；视频点击后应用容器内全屏播放，非仅缩略图。
2. **覆盖范围**：回顾/明悟无图片/视频无需改；需支持全屏预览的页面：**碎屑 + 沉思**。
3. **预览层范围**：应用本身显示范围内全屏（`max-w-md` 容器内），不超设备屏幕；占满整个 `max-w-md` 容器，预览打开时隐藏顶部标题栏和底部 TabBar，适配 9:16/16:9/3:4 等比例。
4. **多图/多视频切换**：支持左右滑动切换同一条记录内多个图片/视频。
5. **缩放**：支持 pinch 缩放；支持双击放大/缩小。
6. **关闭**：点击遮罩 / 点击右上角 × / 电脑端按 ESC。
7. **视频播放**：自动播放、不静音（默认有声）、循环播放。

### 需求 9：设置页展示方式重构

**背景**：当前系统设置是左侧固定菜单 + 右侧内容区的分栏布局（Seam 2 实施）。用户希望改为更紧凑的移动端常见模式：左侧抽屉展示菜单和标签，点击某项后进入全页详情，顶部用横向导航栏切换不同设置项。

**目标**：只改展示方式，不改设置内容。实现抽屉菜单 + 全页设置详情 + 顶部横向导航栏的新交互。**本需求推翻 Seam 2 的左侧分栏布局。**

**已确认**：

1. **左侧抽屉**：点击顶部标题栏 `≡` 设置按钮后，从左侧滑出抽屉。抽屉上半部分展示原左侧设置菜单项（对话模型/语音朗读/向量与语义/数据管理/提示词配置/标签设置/关于）；下半部分新增「所有标签」区块展示现有标签列表；「所有标签」右上角增加快捷入口，点击直接跳转「标签设置」。
2. **点击菜单项跳转**：点击抽屉中某个设置项，不再在右侧展开，而是跳转到全屏设置详情页。全屏详情页顶部增加横向滚动导航栏，包含抽屉里所有设置项；点击横向导航栏项目切换详情。
3. **标签设置入口**：「标签设置」作为顶部横向导航栏一个 tab；同时保留抽屉「所有标签」右上角快捷入口。
4. **桌面端适配**：桌面端宽屏也采用抽屉 + 全页模式，不保留原左右分栏。
5. **抽屉设置项展示**：图标、排序、分组与现在完全一致，不调整。
6. **全屏详情页返回/关闭**：全屏详情页直接关闭设置，不再返回抽屉。
7. **横向导航栏交互**：点击 tab 切换有动画过渡；支持左右滑动切换 tab。
8. **横向导航栏选中高亮**：胶囊/药丸样式高亮当前选中 tab。
9. **标签区块滚动**：标签数量多时仅「所有标签」区块内部滚动，抽屉其他部分不跟随；区块高度根据整体排版自适应，不强制固定比例。

## 跨需求约束（来自 gaps 核对遗漏）

### 多媒体卡片细节（gaps G9/G10）
- **混合媒体布局顺序**：1. 图片/视频 2×2 网格（混排，单格 16:9，最多 4 格，超出 +N） -> 2. 音频纵向列表（每个独立播放器） -> 3. AI 摘要文本。
- **+N 详情内容**：点击 +N 进入该条碎屑的详情/完整视图，含**全部附件 + 完整摘要 + 原始内容**。

### 沉思卡片细节（gaps G7）
- **展开态**：展开后完全展开，**不限制高度**；再次单击收回。

### 提示词合并细节（gaps G6）
- **字段名**：`mingwuInsightPrompt`（明悟/洞察/custom1-3）/ `diaryReviewSummaryPrompt`（日记/回顾摘要合并）/ `mingwuInsightSummaryPrompt`（明悟洞察摘要，由原洞察摘要扩展）。
- **明悟摘要默认值**：补默认明悟摘要 prompt（可用洞察摘要副本作默认值）。

### 无障碍
- 顶部栏按钮保留 `aria-label`。
- 沉思折叠/展开按钮需可被屏幕阅读器识别。
- 附件面板选项需有清晰焦点状态。

### 性能
- 沉思卡片折叠态**不应预渲染完整 Markdown**，可用 `line-clamp` 或最大高度截断。

### 移动端红线
- 全局 `html/body` 保持 `overflow:hidden` + `overscroll-behavior:none`。
- 所有新滚动区（日期选择器/附件面板/设置抽屉/全屏预览/标签区块）用局部 `overflow-y-auto`，不依赖 body 滚动。

### 数据兼容性
- Settings store 版本 v11 -> v12，**在同一 migrate 函数中**协调处理 Prompt 合并 + TTS 外部配置字段初始化。

### 测试
- 组件/单元测试：Vitest + React Testing Library。
- E2E 测试：Playwright/Cypress。
- 视觉回归：对顶部栏/设置页/多媒体卡片/沉思卡片/附件面板做快照对比。

### Out of Scope
- 不改动数据同步、云备份、OAuth 相关逻辑。
- 不涉及端侧 AI（Capacitor/LiteRT-LM）迁移。
