## 需求 1：统计小字下移与统一

### 背景

当前顶部全局 header 显示"白描 · 今日 165 字"作为副标题。各页面内部 header 也重复显示类似统计。信息分散且占用顶部空间。

### 目标

把"今日 X 字"从顶部 header 移到各输入/操作模块的左上方，统一格式为"今日 X 条 X 字"，并清理重复展示。

### 具体改动

1. **顶部全局 header** (`Layout.tsx`)
   - 移除标题旁的"今日 X 字"副标题。

2. **碎屑页** (`Record.tsx`)
   - 移除页面内部 header 里的"今日 X 字"统计 pill。
   - 在底部输入框左上方新增"今日 {count} 条 {chars} 字"。
   - 统计当前查看日期 (`?date=`) 的 `raw_logs`：
     - `count`: 日志条目数。
     - `chars`: `countChars(log.content)`，包含语音/音频转写文本，不包含图片/视频 AI 摘要。

3. **回顾页** (`Review.tsx`)
   - 移除页面内部 header 里的"今日 X 字"统计 pill。
   - 在当前查看日期 (`review_date`) 的日记+回顾卡片列表底部新增"今日 {count} 条 {chars} 字"：
     - 空状态时放在"AI 智能整理"按钮上方。
     - 有内容时放在底部"AI 智能整理(追加)"按钮上方。
   - 统计口径：`countChars(review.ai_editorial || review.ai_review)`。

4. **沉思页** (`Thoughts.tsx`)
   - 移除页面内部 header 里的"总条数 · 总字数"统计。
   - 在底部快速输入/富文本编辑器左上方新增"今日 {count} 条 {chars} 字"。
   - 按**真实今日**(`created_at`) 统计 `thoughts`。

5. **明悟页** (`Mingwu`)
   - 不显示任何"今日 X 条 X 字"统计。

### 样式

- 字号：`text-[11px]`
- 颜色：`text-stone-400`
- 字重：`font-medium`
- 位置：模块左上方，外边距 `mb-1.5`
- 0 条、非今日、按钮禁用时均显示。

### i18n

- 新增通用 key：`common.todayStats`
  - 中文：`今日 {count} 条 {chars} 字`
  - 英文：`Today {count} notes · {chars} chars`

### 实现建议

- 新增共享组件 `src/components/TodayStats.tsx`。
- 数据计算保留在各页面内部，通过 props 传入组件。

## 需求 2：随机漫步屏幕自适应修复

### 背景

随机漫步(`RandomWalk`)是固定全屏覆盖层(`fixed inset-0`)，但卡片区域目前写死为 `w-full max-w-md h-full max-h-[60vh]` 并居中。在桌面宽屏下卡片显得过小、四周留白过多；手机端也可能因 `max-h-[60vh]` 导致空间利用不足。

### 目标

修复随机漫步组件在手机端和桌面端的屏幕适配，让卡片与操作区域在不同屏幕尺寸下都有合理的占用比例。

### 参考实现

- Blinko 每日回顾页面: `D:\DProjects\blinko\app\src\pages\review.tsx`
- Blinko 卡片样式: `D:\DProjects\blinko\app\src\styles\swiper-cards.css`
- Blinko 使用 `swiper` + `EffectCards` 实现扇形卡片堆叠滑动,`swiper` 版本 `^11.2.6`,尺寸:
  - 手机端: `w-[300px] h-[calc(100vh_-_300px)]`
  - 桌面端: `md:w-[550px]`(高度保持 `calc(100vh_-_300px)`)
- 当前白描实现: `src/components/RandomWalk.tsx`

### 已确认

1. **布局模式**
   - 随机漫步改为**主内容区模式**，限制在应用容器(`max-w-md`)内，不再作为全屏覆盖层。
   - 手机端和桌面端都保留顶部 app header（显示"随机漫步"标题，右侧保留 × 关闭按钮）和底部 TabBar（点击 Tab 即退出随机漫步）。

2. **退出方式**
   - 点击顶部 header 灯泡按钮（再次点击关闭）。
   - 点击 header 右侧 × 按钮。
   - 点击底部任意 Tab。

3. **卡片交互**
   - 中间卡片区域采用扇形堆叠滑动效果，使用 `swiper` + `EffectCards`。
   - `swiper` 版本: `^11.2.6`。
   - "下一张"通过左右滑动实现，不再单独放按钮。

4. **底部操作栏**
   - 简化为单排按钮：已阅 / 标签 / 编辑 / 复制 / 删除 / 换一批。
   - 保留底部"标签"按钮和现有 tag sheet，不在编辑弹窗内处理标签。

5. **编辑弹窗**
   - 编辑按钮点击后，所有类型统一弹出 `RichEditor` 编辑弹窗，不跳转页面。
   - 保存时根据记录类型写回对应表和字段，后台自动处理。
   - `raw_logs` / `thoughts` 支持附件编辑；`daily_reviews` / `mingwu` 仅文本。
   - 弹窗含删除按钮，不显示/不修改展示时间。

6. **卡片尺寸**
   - 宽度占满容器宽度，父容器左右 `px-5` padding。
   - 高度由父容器 `flex flex-col`，卡片区域 `flex-1` 自动填充 header 与底部操作栏之间空间。
   - 空状态保持现有居中卡片式，仅限制在应用容器内。
   - 卡片内文字大小保持现有字号，不做响应式调整。

### 实现要点

- 新增依赖:`swiper` `^11.2.6`。
- `Layout.tsx` 中把 `RandomWalk` 从 `fixed` 覆盖层改为在 `<main>` 内容区渲染(类似 `Copilot` 模式),保持顶部 header 和底部 TabBar。
- 新增/复用全局状态控制随机漫步显隐,入口仍为 header 灯泡按钮。
- 随机漫步激活时,顶部 header 标题显示 `randomWalk.title`,右侧显示 × 关闭按钮。
- 底部 TabBar 点击切换 Tab 时退出随机漫步。
- 卡片区域使用 `Swiper` + `EffectCards`,包裹现有 `WalkItem` 卡片内容。
- 编辑按钮改为弹出 `RichEditor` 编辑弹窗,不跳转。
- 标签按钮保留现有 tag sheet。

## 需求 3：碎屑编辑弹窗多媒体化

### 背景

碎屑页 (`Record.tsx`) 的编辑当前使用底部滑上全屏面板 + 纯 `textarea`，不支持多媒体附件的增删。随着 #6 多媒体附件上线，创建时已支持图片/音频/视频/链接/文件，但编辑流程仍停留在纯文本阶段，与产品能力不匹配。

### 目标

把碎屑编辑从底部全屏面板改为居中弹窗，复用 `RichEditor` + 附件编辑能力，让碎屑在创建后仍可追加、删除多媒体附件。

### 具体改动

1. **形态变更**
   - 移除现有底部滑上全屏编辑面板 (`Record.tsx` 中 `isEditingModalOpen` 的旧实现)。
   - 改为居中弹窗，复用沉思编辑弹窗的尺寸规范：
     - 外层：`fixed inset-0 bg-black/40 backdrop-blur-sm flex items-end sm:items-center justify-center p-3`
     - 弹窗：`bg-white rounded-2xl w-full max-w-md max-h-[88vh] flex flex-col`
     - 内容区可滚动，底部操作栏固定。

2. **编辑器**
   - 使用 `RichEditor` 组件，完整保留 Markdown 工具栏、上传、超链接、录音、标签提示、预览切换。
   - 编辑时回显原 `raw_log.attachments` 数组。
   - 原有 `audioBlob`（单独字段）**保留但不可在弹窗内编辑**；它继续随记录存在，删除整条记录时才一并删除。

3. **附件处理**
   - **新增附件**：走与创建时相同的 `saveAttachmentBlob` + 异步摘要生成逻辑。
     - 图片/视频：保存后后台异步调多模态模型生成 `attachment_summary`。
     - 音频：保存后走 STT，转写文本拼入 `content`。
     - 链接/文件：仅保存元数据。
   - **删除已有附件**：从 `attachments` 数组移除，并同步删除 `attachments` store 中对应的 Blob，避免孤儿数据。
   - **清空所有图片/视频附件**：`attachment_summary` 一并清空。

4. **保存条件**
   - 保存按钮启用条件：`content.trim()` 非空 或 `attachments` 非空。
   - 保存时调用 `processTags(editContent)` 重新解析 `#标签` 并落库。

5. **底部操作栏**
   - 左侧：删除按钮（点击确认后删除整条记录并关闭弹窗）。
   - 右侧：取消 / 保存。
   - 字数统计保留在操作栏左侧（使用 `record.totalChars`）。

6. **入口**
   - 保留现有长按/右键菜单的“编辑记录”入口。
   - 与需求 4 联动：双击碎屑卡片也可打开此弹窗。

### i18n

- 复用现有 key：`record.editTitle`、`record.save`、`record.saving`、`record.cancel`、`record.totalChars`、`record.saveFailed`。
- 无需新增 key。

### 实现要点

- 需要让 `RichEditor` 能正确渲染已存附件的缩略图。已存附件的 `ref` 是 `attachments` store 的 id（非 data URL），需在弹窗层加载 Blob 后转成 object URL 传给 `RichEditor`，或在 `RichEditor` 内部增加对 store id 的解析能力。
- 新增附件由 `RichEditor` 内部读 File 为 data URL；保存时区分“新附件（data URL / File）”和“旧附件（store id）”，仅对新附件调 `saveAttachmentBlob`。
- 删除附件时，对旧附件调 `db.attachments.delete(ref)`。

## 需求 4：碎屑/回顾卡片双击编辑

### 背景

目前编辑碎屑和回顾都需要通过长按/右键菜单或展开后点编辑按钮，操作路径较长。用户希望像桌面端习惯一样，双击内容卡片即可直接进入编辑状态。

### 目标

给碎屑页和回顾页的内容卡片增加双击编辑能力，缩短编辑路径。

### 已确认

- **碎屑页**：双击卡片任意区域打开需求 3 的新编辑弹窗，弹窗内所有字段可编辑。
- **回顾页**：维持现有 inline 编辑（卡片展开后内部变成 textarea），双击后触发该 inline 编辑状态。
- **单击**：回顾卡片保持展开/折叠；碎屑卡片保持现有单击行为。
- **多选模式**：启用多选时禁用双击编辑。

### 具体改动

1. **碎屑页 (`Record.tsx`)**
   - 给 `LogCard` 绑定 `onDoubleClick`（卡片任意区域，除操作按钮外）。
   - 双击直接打开需求 3 的 `RichEditor` 编辑弹窗。
   - 多选模式下不响应双击。
   - 附件区单击继续保留打开灯箱/详情的现有行为；只在非附件元素上触发双击编辑。

2. **回顾页 (`Review.tsx`)**
   - 给回顾内容卡片绑定 `onDoubleClick`。
   - 双击触发现有 inline 编辑状态（即展开后 textarea 直接获得焦点）。
   - 多选模式下不响应双击。

### 实现要点

- 通过 `onDoubleClick` 事件实现，不额外引入双击库。
- 需要处理好单击与双击在同一元素上的互斥：回顾页单击折叠/展开，双击进编辑。
- 多选状态由父组件统一控制，进入多选后移除双击监听。

   - 维持现有高度不变。

## 需求 7：图片/视频全屏预览适配修复

### 背景

当前图片虽然可以点击打开，但预览层没有做好屏幕自适应，会突破应用容器变成真正的全屏覆盖；视频则只能在卡片缩略图内播放，没有放大到屏幕大小的预览能力。

### 目标

统一图片与视频的点击预览体验：点击缩略图后在应用界面内（`max-w-md` 容器）全屏打开，并做好自适应，不出现容器外溢。

### 已确认

1. **核心行为**
   - 图片点击后应在应用容器内全屏预览，不再超出 `max-w-md` 边界。
   - 视频点击后应在应用容器内全屏播放，而不是仅在缩略图里播放。

2. **覆盖范围**
   - 回顾、明悟页面没有图片/视频，无需改动。
   - 需要支持图片/视频全屏预览的页面：**碎屑 + 沉思**。

3. **预览层范围**
   - 在应用本身显示范围内全屏，即限定在 `max-w-md` 容器内，不再超出到整个设备屏幕。
   - 占满整个 `max-w-md` 应用容器，预览打开时隐藏顶部标题栏和底部 TabBar，以便适配 9:16、16:9、3:4 等多种比例。

4. **多图/多视频切换**
   - 支持左右滑动切换同一条记录内的多个图片/视频。

5. **缩放能力**
   - 支持 pinch 缩放。
   - 支持双击放大/缩小。

6. **关闭方式**
   - 点击遮罩关闭。
   - 点击右上角 × 关闭。
   - 电脑端按 ESC 关闭。

7. **视频播放设置**
   - 自动播放。
   - 不静音（默认有声）。
   - 循环播放。

## 需求 8：碎屑附件上传入口动画面板

### 背景

碎屑页底部输入框的附件上传按钮（回形针）目前点击后可能是直接调起系统文件选择器，缺少一个中间面板让用户选择上传类型。参考微信聊天输入，点击 + 号后从底部滑出一块功能面板，体验更直观。

### 目标

点击碎屑输入框旁的附件按钮后，以动画形式展开底部附件选择面板，面板内列出可用的上传类型。

### 已确认

1. **触发入口**
   - 碎屑页底部输入框旁的回形针/附件按钮。

2. **面板内容**
   - 最终展示状态参考微信聊天输入底部面板：从底部滑出，网格排列功能图标。
   - 面板内容沿用现有选项，不做增减：
     - 相册
     - 音频
     - 视频
     - 链接
     - 文件
     - 取消

3. **面板展开方式**
   - 面板从底部滑出，把输入栏和上方主内容一起向上推。

4. **关闭方式**
   - 点击面板中的「取消」按钮。
   - 点击面板外部区域。
   - 下滑手势。
   - 多种关闭方式并存。

5. **使用范围**
   - 仅在碎屑页使用。

6. **动画参数**
   - 动画时长和缓动曲线按系统默认。

## 需求 9：设置页展示方式重构

### 背景

当前系统设置是左侧固定菜单 + 右侧内容区的分栏布局。用户希望改为更紧凑的移动端常见模式：左侧抽屉展示菜单和标签，点击某一项后进入全页详情，顶部用横向导航栏切换不同设置项。

### 目标

只改展示方式，不改设置内容。实现抽屉菜单 + 全页设置详情 + 顶部横向导航栏的新交互。

### 已确认

1. **左侧抽屉**
   - 点击顶部标题栏的 `≡` 设置按钮后，从左侧滑出抽屉。
   - 抽屉上半部分展示原来的左侧设置菜单项（如：对话模型、语音朗读、向量与语义、数据管理、提示词配置、标签设置、关于）。
   - 抽屉下半部分新增「所有标签」区块，展示现有标签列表。
   - 「所有标签」区块右上角增加快捷入口，点击直接跳转到「标签设置」页面。

2. **点击菜单项后的跳转逻辑**
   - 点击抽屉中的某个设置项（如「对话模型」），不再在右侧展开，而是跳转到全屏设置详情页。
   - 全屏详情页顶部增加横向滚动导航栏，包含之前左侧抽屉里的所有设置项。
   - 点击横向导航栏中的项目，切换到对应设置详情。

3. **标签设置入口**
   - 「标签设置」作为顶部横向导航栏中的一个 tab。
   - 同时保留抽屉中「所有标签」右上角的快捷入口，点击也进入标签设置。

4. **桌面端适配**
   - 桌面端宽屏也采用抽屉 + 全页模式，不保留原来的左右分栏。

5. **抽屉设置项展示**
   - 图标、排序、分组与现在完全一致，不做调整。

6. **全屏详情页返回/关闭逻辑**
   - 全屏详情页直接关闭设置，不再返回抽屉。

7. **横向导航栏交互**
   - 点击 tab 切换时有动画过渡。
   - 支持左右滑动切换 tab。

8. **横向导航栏选中高亮**
   - 采用胶囊/药丸样式高亮当前选中的 tab。

9. **标签区块滚动**
   - 标签数量多时，仅「所有标签」区块内部滚动，抽屉其他部分不跟随滚动。
   - 区块高度根据整体排版自适应，不强制固定比例。
