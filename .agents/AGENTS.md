# 白描笔记 — 项目级 Agent 规则

## 重命名安全检查

在对任何变量、函数、类型进行重命名后，必须执行以下步骤，否则不得提交：

1. 使用 `grep_search` 在整个相关目录中搜索**旧名称**，确认结果为零
2. 如果搜索到残留引用，逐一修复
3. 在 commit message 或改动说明中注明「已通过 grep 确认旧名称零残留」

此规则适用于所有重构场景，包括但不限于：
- 状态变量重命名（如 React useState）
- 函数/方法重命名
- 接口/类型重命名
- CSS 类名重命名

## 命名约定异常：保留产品标识符

尽管全局规则要求“不使用拼音”，但对于本项目的核心产品名称 **“白描” (baimiao)** 属于绝对例外：
- **禁止去拼音化**：在任何变量名、Storage 键名、API 路由或本地路径中（如 `baimiao_` 键前缀、`/baimiaobiji/` 目录），绝对禁止将 `baimiao` 翻译或重命名为 `whitewash` 或其他英文单词。
- `baimiao` 必须作为一个合法的专用名词在整个代码库中予以保留。

## 多模板生成存在性检查规约

由于白描笔记支持用户配置并同时启用多个不同的日记/回顾 Prompt 模板，在任何涉及日记（`daily_diaries`）和回顾（`daily_reviews`）的“已存在校验”、“生成检测”或“自动补全”逻辑中，必须遵守以下要求：

1. **禁止仅通过日期进行“有无”判断**：
   绝对禁止在查询到该日期已有一篇日记/回顾（例如使用 `.first()` 判空）后就直接忽略或退出。这会导致用户启用多 Prompt 模板时漏掉其它模板内容的生成。
2. **必须根据 Prompt 索引精确匹配**：
   必须提取当前所有启用的 Prompt 索引列表（如通过 `getActivePromptIndices`），与数据库中该日期下已存储数据记录的 `prompt_index` / `review_prompt_index` 集合进行差集对比，精确盘点具体是哪一个模板的数据，进而发起增量补全。

## 经典修长仿宋体垂直对齐规约

当项目需要清秀、修长且完全开源商用免版权的中文字体时，应当优先选用经典的**仿宋体（FangSong）**家族：

- **CSS 字体声明**：`font-family: FangSong, STFangsong, "华文仿宋", serif;`
- **视觉重心下移补偿**：由于仿宋体字身修长、笔画纤细，其视觉几何重心相对于普通的加粗黑体字天然略微偏上。在包含搜索、设置等动作图标的 Flex 垂直居中容器（`flex items-center`）中，**必须显式添加 `translate-y-[2px]` 进行物理下移补偿**，才能使 Logo 等文本与右侧按钮图标在视网膜上达成像素级的绝对垂直平齐。

## 文本卡片防抖动与抗虚化动效规约

对于包含细密文本或小字号（如 <16px）正文的交互气泡与卡片组件（例如 `.baimiao-card-bubble`），在定义其悬浮（Hover）与激活（Active）过渡效果时，必须遵循以下防渲染缺陷原则：

- **避免 Hover 物理位移**：在 Hover 态下**禁止使用 `transform: translateY`** 等会造成元素在物理空间产生位移的转换属性。位移动画会强迫浏览器进行合成层（Composited Layer）切换，在动画结束的一瞬间会因重新栅格化与抗锯齿切换产生肉眼可见的“文字变胖、变细、闪烁或虚化抖动”。
- **纯粹的光学漫射反馈**：推荐仅通过 `box-shadow`（如暮光紫渐变投影）和 `border-color`（如边框变淡紫）的过渡来提供悬浮反馈：
  `transition: box-shadow 0.3s ease, border-color 0.3s ease !important;`
- **按压反馈降级**：在移动端（`hover:none`）的按压反馈上，同样避免对包含密集文本的卡片使用 `scale` 缩放，推荐通过微调背景色与边框色深浅（如变更为 `bg-[#f6f4f9]`）来提供温润稳定的触感回馈。
